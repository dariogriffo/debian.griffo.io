Was ist Fastfetch?

Fastfetch ist ein neofetch-ähnliches Tool zum Abrufen von Systeminformationen, das diese ansprechend in Ihrem Terminal darstellt — OS, Kernel, Hardware, Uptime, Pakete, Desktop-Umgebung und vieles mehr, neben dem Logo Ihrer Distribution. Hauptsächlich in C geschrieben und auf Performance und Anpassbarkeit ausgelegt, wurde es zum De-facto-Nachfolger, als neofetch archiviert wurde, und wird mit häufigen Releases aktiv weiterentwickelt.

😤 Das Archiv-Versions-Problem: Debian liefert fastfetch zwar mit — eingefroren auf der Version, die beim Release existierte. Upstream entwickelt sich schnell: neue Logos, neue Erkennungsmodule und Fixes für neue Hardware kommen ständig hinzu, sodass die Archiv-Version still zurückfällt. Dieses Repository liefert aktuelle Upstream-Builds und behält dabei exakt dasselbe Datei-Layout wie das offizielle Debian-Paket, sodass der Wechsel in beide Richtungen nahtlos ist.
🆓 Für immer kostenlos, mit eingebauter Verzögerung: Debian und Ubuntu frieren Paketversionen mit jedem Release ein — dieses Repository liefert stattdessen das jeweils neueste Release, sobald es mindestens 2 Monate alt ist, für immer kostenlos; Sicherheits- und Patch-Fixes erscheinen sofort. Wer jedes Release wenige Stunden nach Upstream möchte, nutzt das Haupt-Repository auf deb.griffo.io.

⚡ Hauptfunktionen von Fastfetch

🏎️ Schnell durch Design

Geschrieben in C mit nebenläufiger Erkennung — rendert vollständige Systeminformationen in Millisekunden, ein Bruchteil der Startzeit von neofetch.

🧩 50+ Info-Module

OS, Kernel, CPU, GPU, Arbeitsspeicher, Festplatten, Akku, Display, DE/WM, Terminal, Pakete, Media-Player und viele mehr — jedes einzeln konfigurierbar.

🎨 Presets enthalten

Dieses Paket liefert alle integrierten Presets unter /usr/share/fastfetch/presets/ — inklusive eines originalgetreuen neofetch-Presets für sofortige Vertrautheit.

📝 JSONC-Konfiguration

Eine dokumentierte JSONC-Konfigurationsdatei steuert Module, Formatierung und Logos. Erzeugen Sie einen Startpunkt mit fastfetch --gen-config.

🖼️ Flexible Logos

Hunderte integrierte Distro-Logos, dazu Bild-Logos über sixel/kitty/iTerm-Protokolle, eigene ASCII-Art oder gar kein Logo.

🐚 Shell-Completions

Bash-, zsh- und fish-Completions plus Man-Page — installiert in die Standard-Debian-Pfade, passend zum Archiv-Paket.

🏆 Warum dieses Repository statt der Archiv-Version?

  • Aktuelle Version: neuestes Release, das mindestens 2 Monate alt ist — nicht für die Lebensdauer eines Debian-Releases eingefroren
  • Gleiches Layout: Datei für Datei identisch mit dem offiziellen Debian-Paket — Presets, Completions und Man-Page an den Standardorten
  • Nahtloser Wechsel: die Installation aus diesem Repo aktualisiert einfach das Archiv-Paket; nach Entfernen des Repos fällt alles sauber zurück
  • Automatische Updates: neue Releases kommen mit Ihrem normalen apt upgrade
  • Multi-Arch: amd64, arm64, armel, armhf, ppc64el, s390x, riscv64 und i386

❓ Häufig gestellte Fragen

Ist Fastfetch in den offiziellen Debian-Paketquellen?

Ja — das offizielle Debian-Archiv liefert Fastfetch 2.62.1. Dieser kostenlose Spiegel liefert das neueste Fastfetch-Release, das mindestens 2 Monate alt ist, für immer kostenlos; das Haupt-Repository auf deb.griffo.io liefert jedes Release wenige Stunden nach Upstream.

Wie installiere ich das neueste Fastfetch unter Debian?

Fügen Sie das deb-free.griffo.io-Repository einmalig hinzu (Anleitung oben) und führen Sie dann aus: sudo apt install fastfetch. Neue Releases kommen über das normale sudo apt upgrade.

Sind die Pakete signiert und wie werden sie gebaut?

Jedes Paket ist mit dem GPG-Schlüssel des Repositories (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) signiert und wird aus Upstream-Releases in öffentlichen GitHub-Repositories gebaut, die jeder einsehen kann.
