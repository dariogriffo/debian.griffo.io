Was ist Ruff?

Ruff ist ein extrem schneller Python-Linter und Code-Formatter, geschrieben in Rust von Astral — dem Team hinter uv. Mit über 800 integrierten Regeln und nativen Neuimplementierungen beliebter Plugins ersetzt ein einziges ruff-Binary Flake8, isort, Black, pyupgrade, autoflake und Dutzende Plugins — und läuft dabei 10–100× schneller. Es ist zum Standard-Linter des modernen Python-Ökosystems geworden, eingesetzt von Projekten wie FastAPI, Pandas und Hugging Face.

😤 Typische Python-Tooling-Probleme: Sie verwalten flake8 plus ein Dutzend Plugins, isort streitet mit Black, Lint-Läufe dauern in CI Minuten, und Linter müssen in jedes einzelne virtualenv installiert werden? Ruff ist ein einziges abhängigkeitsfreies Binary, das alles in Millisekunden erledigt — und mit apt installiert ist es systemweit verfügbar, für jedes Projekt und jedes venv, ganz ohne pip.
🆓 Für immer kostenlos, mit eingebauter Verzögerung: Debian und Ubuntu frieren Paketversionen mit jedem Release ein — dieses Repository liefert stattdessen das jeweils neueste Release, sobald es mindestens 2 Monate alt ist, für immer kostenlos; Sicherheits- und Patch-Fixes erscheinen sofort. Wer jedes Release wenige Stunden nach Upstream möchte, nutzt das Haupt-Repository auf deb.griffo.io.

⚡ Hauptfunktionen von Ruff

🏎️ 10–100× schneller

Lintet große Codebasen in Millisekunden, wo Legacy-Tools Minuten brauchen — schnell genug für jeden Speichervorgang und jede CI-Pipeline.

🧰 Ein Tool, 800+ Regeln

Reimplementiert Flake8, isort, pyupgrade, pydocstyle, eradicate und mehr — eine Konfiguration in pyproject.toml ersetzt einen ganzen Plugin-Stack.

🖤 Drop-in-Formatter

ruff format ist ein Black-kompatibler Formatter — derselbe Stil, um Größenordnungen schneller, kein separates Tool zu installieren.

🔧 Auto-Fixes

Hunderte Regeln bringen automatische Korrekturen mit: ungenutzte Imports, modernisierte Syntax, sortierte Imports — ruff check --fix und fertig.

🐍 Versionsbewusst

Unterstützt jedes Python ab 3.7 mit versionsspezifischem Regelverhalten, monorepo-freundlicher hierarchischer Konfiguration und Überschreibungen pro Datei.

🔌 Editor-Integrationen

Ein integrierter Language Server (ruff server) liefert erstklassige Integrationen für VS Code, Zed, Neovim und Helix aus demselben Binary.

🏆 Warum mit apt installieren?

  • Systemweit, ohne pip: ein Binary für jedes Projekt, jedes venv, jeden CI-Job und jeden Nutzer — keine Installationen pro Projekt, die synchron gehalten werden müssen
  • Perfekt für Docker/CI-Images: das Repo in Basis-Images einbauen, und jede Pipeline erhält automatisch aktuelles ruff
  • Keine Toolchain: kein Rust, kein cargo, kein pip — ein signiertes apt-Repository mit vorgebauten Binaries für 8 Architekturen
  • Automatische Updates: ruff veröffentlicht häufig neue Versionen; apt upgrade hält Schritt, damit Ihre Regeln nie veralten
  • Passt zu uv: Astrals Paketmanager wird ebenfalls hier angeboten — die komplette moderne Python-Toolchain aus einem Repository

❓ Häufig gestellte Fragen

Ist Ruff in den offiziellen Ubuntu-Paketquellen?

Nein — Ruff ist nicht in den offiziellen Ubuntu-Archiven paketiert. Dieser kostenlose Spiegel liefert das neueste Ruff-Release, das mindestens 2 Monate alt ist; das Haupt-Repository auf deb.griffo.io liefert jedes Release wenige Stunden nach Upstream.

Wie installiere ich das neueste Ruff unter Ubuntu?

Fügen Sie das deb-free.griffo.io-Repository einmalig hinzu (Anleitung oben) und führen Sie dann aus: sudo apt install ruff. Neue Releases kommen über das normale sudo apt upgrade.

Sind die Pakete signiert und wie werden sie gebaut?

Jedes Paket ist mit dem GPG-Schlüssel des Repositories (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) signiert und wird aus Upstream-Releases in öffentlichen GitHub-Repositories gebaut, die jeder einsehen kann.
