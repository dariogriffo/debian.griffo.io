O que é o yq?

O yq é um processador de linha de comando leve e portátil para dados estruturados — arquivos YAML, JSON, XML, CSV, TOML e properties — escrito em Go por Mike Farah. Ele usa sintaxe no estilo do jq, então tudo o que você sabe do jq se transfere diretamente: consulte, transforme, mescle e edite arquivos no local, em scripts e pipelines, com um único binário sem dependências.

⚠️ Qual yq é este? Este é o mikefarah/yq (v4.x, Go) — o usado na maioria dos exemplos de CI e documentações. Não é o wrapper de jq yq baseado em python encontrado em alguns arquivos; este pacote o substitui de forma limpa, e nenhum python ou jq é necessário.
🚀 Sempre à frente dos repositórios oficiais: Debian e Ubuntu congelam as versões dos pacotes a cada lançamento — este repositório publica novas versões upstream normalmente em poucas horas. Um simples apt upgrade mantém você sempre na versão mais recente.

⚡ Principais Recursos do yq

🔄 Seis Formatos

YAML, JSON, XML, CSV/TSV, TOML e properties — leia um, produza outro: yq -o=json '.' config.yaml converte em um único passo.

🧠 Sintaxe no Estilo do jq

Caminhos, pipes, funções e operadores familiares. Se você sabe escrever um filtro jq, já sabe usar o yq em YAML.

✏️ Edição no Local

yq -i '.spec.replicas = 3' deploy.yaml — edições via script em manifestos do Kubernetes, configurações de CI e arquivos Compose, com comentários preservados.

📦 Zero Dependências

Um único binário estático em Go. Sem runtime, sem bibliotecas, sem python — ideal para contêineres mínimos e runners de CI.

🔀 Merge e Âncoras

Faz merge profundo de múltiplos arquivos, resolve âncoras e aliases de YAML, lida com fluxos de múltiplos documentos — os casos difíceis de YAML feitos do jeito certo.

📖 Página de Manual Incluída

Este pacote traz a página de manual completa junto com o binário — man yq funciona offline, seguindo as convenções do Debian.

🏆 Por que instalar com o apt?

  • Amigável a frotas: fixe um repositório nas suas imagens base e cada servidor, contêiner e runner de CI recebe o mesmo yq atual
  • Sem curl-para-bin: um repositório assinado com GPG em vez de baixar binários manualmente para /usr/local/bin
  • O yq certo: garante a semântica do mikefarah/yq v4 em todo lugar — sem surpresas do wrapper em python em algumas máquinas
  • Atualizações automáticas: novos lançamentos upstream chegam com o seu apt upgrade de sempre
  • Multi-arquitetura: amd64, arm64, armhf, ppc64el, s390x e riscv64

❓ Perguntas frequentes

O yq está nos repositórios oficiais do Ubuntu?

Não — yq não está empacotado nos arquivos oficiais do Ubuntu. Este espelho gratuito oferece a versão mais recente do yq com pelo menos 2 meses; o repositório principal em deb.griffo.io oferece cada versão poucas horas após o upstream.

Como instalo o yq mais recente no Ubuntu?

Adicione o repositório deb-free.griffo.io uma única vez com as instruções acima e execute: sudo apt install yq. Novos lançamentos chegam pelo sudo apt upgrade normal.

Os pacotes são assinados e como são construídos?

Cada pacote é assinado com a chave GPG do repositório (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) e construído a partir de lançamentos upstream em repositórios públicos do GitHub que qualquer pessoa pode inspecionar.

Quais versões do Ubuntu são suportadas?

Ubuntu 22.04 Jammy, 24.04 Noble, 25.10 Questing e 26.04 Resolute.
