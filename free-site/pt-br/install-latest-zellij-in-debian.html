O que é o Zellij?

O Zellij é um workspace de terminal e multiplexador escrito em Rust. Uma alternativa moderna ao tmux e ao screen, ele permite dividir seu terminal em painéis e abas, salvar e restaurar layouts, e estendê-lo com um sistema de plugins WebAssembly — tudo isso com atalhos de teclado detectáveis e uma interface amigável que facilitam o aprendizado.

🚀 Por que as últimas versões são importantes: O Zellij é desenvolvido ativamente com novos recursos de layout, capacidades de plugins e melhorias de desempenho. Executar a versão mais recente garante os recursos e correções mais novos do workspace.

⚡ Principais Recursos do Zellij

🪟 Painéis & Abas

Divida seu terminal em painéis e organize o trabalho em várias abas.

📐 Layouts

Defina e restaure layouts de workspace reproduzíveis para seus projetos.

🔌 Sistema de Plugins

Estenda o Zellij com plugins WebAssembly para funcionalidades personalizadas.

🧭 Interface Detectável

Dicas na tela facilitam o aprendizado e a memorização dos atalhos de teclado.

🎯 Por que o Zellij?

  • Amigável para iniciantes: Dicas de atalhos na tela reduzem a curva de aprendizado
  • Layouts poderosos: Workspaces reproduzíveis definidos em configuração
  • Extensível: Sistema de plugins WebAssembly
  • Moderno: Implementação rápida em Rust com padrões sensatos

🆕 Novidades

  • Multiplexação: Painéis e abas em um único terminal
  • Gerenciamento de Sessões: Desconecte e reconecte a sessões em execução
  • Layouts: Salve e carregue arranjos de workspace
  • Plugins: Extensibilidade baseada em WASM

❓ Perguntas frequentes

O Zellij está nos repositórios oficiais do Debian?

Não — Zellij não está empacotado nos arquivos oficiais do Debian. Este espelho gratuito oferece a versão mais recente do Zellij com pelo menos 2 meses; o repositório principal em deb.griffo.io oferece cada versão poucas horas após o upstream.

Como instalo o Zellij mais recente no Debian?

Adicione o repositório deb-free.griffo.io uma única vez com as instruções acima e execute: sudo apt install zellij. Novos lançamentos chegam pelo sudo apt upgrade normal.

Os pacotes são assinados e como são construídos?

Cada pacote é assinado com a chave GPG do repositório (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) e construído a partir de lançamentos upstream em repositórios públicos do GitHub que qualquer pessoa pode inspecionar.

Quais versões do Debian são suportadas?

Debian 12 Bookworm, Debian 13 Trixie, Forky e Sid.
