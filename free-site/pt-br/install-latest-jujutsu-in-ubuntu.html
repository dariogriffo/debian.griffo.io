O que é o Jujutsu?

O Jujutsu é um sistema de controle de versão compatível com Git que oferece um modelo mais simples e poderoso do que o Git, mantendo compatibilidade retroativa com repositórios Git existentes. Ele usa o comando jj, trata a cópia de trabalho como um commit, oferece tratamento de conflitos de primeira classe e desfazer para toda operação — permitindo que você continue usando serviços de hospedagem Git como GitHub ou Forgejo por baixo.

🚀 Por que as últimas versões são importantes: O Jujutsu está evoluindo rapidamente, com novos comandos, refinamentos de fluxo de trabalho e melhorias de desempenho. Usar a versão mais recente significa acesso aos recursos mais novos e correções de interoperabilidade com o Git.

⚡ Principais Recursos do Jujutsu

🔄 Compatível com Git

Funciona diretamente com repositórios Git e remotes existentes, como GitHub e Forgejo.

↩️ Desfazer Universal

Toda operação é registrada e pode ser desfeita com um único comando.

🧩 Conflitos de Primeira Classe

Conflitos são registrados nos commits e podem ser resolvidos a qualquer momento, não apenas durante merges.

📄 Cópia de Trabalho como Commit

A cópia de trabalho é, ela mesma, um commit, eliminando a staging area e simplificando os fluxos de trabalho.

🎯 Por que o Jujutsu?

  • Modelo mais simples: Sem staging area e um fluxo de trabalho consistente e previsível
  • Experimentação segura: Desfaça qualquer operação, incluindo rebases e merges
  • Interoperabilidade com Git: Continue usando hospedagem Git e colabore com usuários Git
  • Edição de histórico poderosa: Rebase e reescrita de commits sem esforço

🆕 Destaques

  • O comando jj: Uma CLI moderna sobre repositórios Git
  • Log de Operações: Histórico completo das operações do repositório, com desfazer
  • Tratamento de Conflitos: Conflitos armazenados em commits, resolvíveis a qualquer momento
  • Backend Git: Interopera com remotes Git existentes

❓ Perguntas frequentes

O Jujutsu está nos repositórios oficiais do Ubuntu?

Não — Jujutsu não está empacotado nos arquivos oficiais do Ubuntu. Este espelho gratuito oferece a versão mais recente do Jujutsu com pelo menos 2 meses; o repositório principal em deb.griffo.io oferece cada versão poucas horas após o upstream.

Como instalo o Jujutsu mais recente no Ubuntu?

Adicione o repositório deb-free.griffo.io uma única vez com as instruções acima e execute: sudo apt install jujutsu. Novos lançamentos chegam pelo sudo apt upgrade normal.

Os pacotes são assinados e como são construídos?

Cada pacote é assinado com a chave GPG do repositório (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) e construído a partir de lançamentos upstream em repositórios públicos do GitHub que qualquer pessoa pode inspecionar.

Quais versões do Ubuntu são suportadas?

Ubuntu 22.04 Jammy, 24.04 Noble, 25.10 Questing e 26.04 Resolute.
