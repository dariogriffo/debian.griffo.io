Qu'est-ce que yq ?

yq est un processeur en ligne de commande léger et portable pour les données structurées — fichiers YAML, JSON, XML, CSV, TOML et properties — écrit en Go par Mike Farah. Il utilise une syntaxe façon jq, donc tout ce que vous connaissez de jq se transfère directement : interrogez, transformez, fusionnez et modifiez des fichiers en place, dans vos scripts et vos pipelines, avec un unique binaire sans dépendances.

⚠️ De quel yq s'agit-il ? Il s'agit de mikefarah/yq (v4.x, Go) — celui utilisé dans la plupart des exemples CI et de la documentation. Ce n'est pas le wrapper jq yq en python présent dans certaines archives ; ce paquet le remplace proprement, et ni python ni jq ne sont requis.
🚀 Gratuit pour toujours : Debian et Ubuntu figent les versions des paquets à chaque publication — ce miroir gratuit propose la version la plus récente de chaque paquet dès qu'elle date d'au moins 2 mois, avec des correctifs de sécurité immédiats. Le dépôt principal sur deb.griffo.io propose chaque version quelques heures après l'upstream.

⚡ Fonctionnalités clés de yq

🔄 Six formats

YAML, JSON, XML, CSV/TSV, TOML et properties — lisez un format, produisez-en un autre : yq -o=json '.' config.yaml convertit en une seule étape.

🧠 Syntaxe façon jq

Chemins, pipes, fonctions et opérateurs familiers. Si vous savez écrire un filtre jq, vous savez déjà utiliser yq sur du YAML.

✏️ Édition en place

yq -i '.spec.replicas = 3' deploy.yaml — des modifications scriptées des manifestes Kubernetes, des configs CI et des fichiers Compose, commentaires préservés.

📦 Zéro dépendance

Un unique binaire Go statique. Pas de runtime, pas de bibliothèques, pas de python — idéal pour les conteneurs minimaux et les runners CI.

🔀 Fusion & ancres

Fusionne en profondeur plusieurs fichiers, résout les ancres et alias YAML, gère les flux multi-documents — les cas YAML difficiles traités correctement.

📖 Page man incluse

Ce paquet fournit la page man complète avec le binaire — man yq fonctionne hors ligne, conformément aux conventions Debian.

🏆 Pourquoi l'installer avec apt ?

  • Pensé pour les flottes : épinglez un seul dépôt dans vos images de base et chaque serveur, conteneur et runner CI obtient le même yq à jour
  • Pas de curl-vers-bin : un dépôt signé GPG au lieu de télécharger des binaires à la main dans /usr/local/bin
  • Le bon yq : garantit la sémantique mikefarah/yq v4 partout — pas de surprises avec le wrapper python sur certaines machines
  • Mises à jour automatiques : les nouvelles versions upstream arrivent avec votre apt upgrade habituel
  • Multi-arch : amd64, arm64, armhf, ppc64el, s390x et riscv64

❓ Questions fréquentes

yq est-il dans les dépôts officiels de Ubuntu ?

Non — yq n'est pas empaqueté dans les archives officielles de Ubuntu. Ce miroir gratuit propose la version la plus récente de yq datant d'au moins 2 mois ; le dépôt principal sur deb.griffo.io propose chaque version quelques heures après l'upstream.

Comment installer le dernier yq sur Ubuntu ?

Ajoutez le dépôt deb.griffo.io une seule fois (instructions ci-dessus), puis exécutez : sudo apt install yq. Les nouvelles versions arrivent via le sudo apt upgrade habituel.

Les paquets sont-ils signés et comment sont-ils construits ?

Chaque paquet est signé avec la clé GPG du dépôt (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) et construit à partir des versions upstream dans des dépôts GitHub publics que chacun peut inspecter.

Quelles versions de Ubuntu sont prises en charge ?

Ubuntu 22.04 Jammy, 24.04 Noble, 25.10 Questing et 26.04 Resolute.
