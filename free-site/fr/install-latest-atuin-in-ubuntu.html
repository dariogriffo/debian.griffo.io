Qu'est-ce qu'Atuin ?

Atuin remplace votre historique shell par une base de données SQLite consultable, en ajoutant une recherche plein texte puissante, une synchronisation chiffrée de bout en bout optionnelle entre plusieurs machines, et des statistiques utiles. Il enregistre un contexte supplémentaire pour chaque commande — code de sortie, durée, répertoire et plus encore — et fonctionne avec Bash, Zsh, Fish et d'autres shells.

🚀 Pourquoi les dernières versions sont importantes : Atuin est activement développé, avec des améliorations constantes de la recherche, de la synchronisation et de l'interface interactive. Utiliser la dernière version vous donne accès aux fonctionnalités et correctifs d'historique les plus récents.

⚡ Fonctionnalités clés d'Atuin

🔍 Recherche puissante

Recherche plein texte, floue et par préfixe sur l'ensemble de votre historique shell.

🔐 Synchronisation chiffrée

Synchronisation chiffrée de bout en bout, optionnelle, de l'historique entre toutes vos machines.

📊 Contexte riche

Enregistre le code de sortie, la durée, le répertoire et plus encore pour chaque commande.

🐚 Multi-shell

Fonctionne avec Bash, Zsh, Fish et d'autres shells populaires.

  • Un meilleur historique : Historique SQLite consultable qui ne se tronque jamais silencieusement
  • Synchronisation entre machines : Synchronisation chiffrée de bout en bout, auto-hébergeable

  • Statistiques : Un aperçu de vos commandes les plus utilisées

❓ Questions fréquentes

Atuin est-il dans les dépôts officiels de Ubuntu ?

Oui — l'archive officielle de Ubuntu fournit Atuin 18.8.0. Ce miroir gratuit propose la version la plus récente de Atuin datant d'au moins 2 mois, gratuitement pour toujours ; le dépôt principal sur deb.griffo.io propose chaque version quelques heures après l'upstream.

Comment installer le dernier Atuin sur Ubuntu ?

Ajoutez le dépôt deb-free.griffo.io une seule fois (instructions ci-dessus), puis exécutez : sudo apt install atuin. Les nouvelles versions arrivent via le sudo apt upgrade habituel.

Les paquets sont-ils signés et comment sont-ils construits ?

Chaque paquet est signé avec la clé GPG du dépôt (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) et construit à partir des versions upstream dans des dépôts GitHub publics que chacun peut inspecter.

Quelles versions de Ubuntu sont prises en charge ?

Ubuntu 22.04 Jammy, 24.04 Noble, 25.10 Questing et 26.04 Resolute.
