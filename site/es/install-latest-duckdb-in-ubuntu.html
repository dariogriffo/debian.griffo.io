¿Qué es DuckDB?

DuckDB es una base de datos SQL analítica, rápida e in-process — piensa en "SQLite para analítica". Se ejecuta dentro del proceso que la llama sin servidor que configurar, consulta archivos CSV, Parquet y JSON directamente (SELECT * FROM 'data.parquet'), y habla un dialecto SQL rico y amigable con un rendimiento excelente en cargas de trabajo analíticas. Este paquete incluye la CLI oficial duckdb.

⚠️ Sin servidor, sin demonio: DuckDB es una base de datos embebida — se ejecuta dentro de tu proceso (o de la CLI), así que no hay servicio que configurar, ni puerto que abrir, ni unidad de systemd. Instálala y ejecuta duckdb. Por la misma razón, un archivo de base de datos pertenece a un solo proceso a la vez.
🚀 Siempre por delante de los archivos oficiales: Debian y Ubuntu congelan las versiones de los paquetes al publicar cada versión — este repositorio publica las nuevas versiones upstream normalmente en cuestión de horas, para todas las versiones soportadas incluidas las LTS. Un simple apt upgrade te mantiene siempre en la última versión.

⚡ Características principales de DuckDB

📊 Hecha para analítica

Un motor columnar y vectorizado procesa millones de filas por segundo — agregaciones, joins y funciones de ventana a velocidad OLAP.

📁 Consulta archivos directamente

SELECT * FROM 'events.parquet' — CSV, Parquet y JSON se pueden consultar (y escribir) sin ningún paso de importación.

🪶 Cero configuración

In-process como SQLite: un único binario, un único archivo de base de datos opcional, nada que administrar.

🔌 Extensiones en tiempo de ejecución

INSTALL httpfs; descarga extensiones firmadas desde el repositorio oficial — acceso S3/HTTP, espacial, búsqueda de texto completo y más.

🗣️ SQL amigable

SELECT * EXCLUDE, GROUP BY ALL, tipos list/struct, PIVOT — un dialecto moderno que elimina el boilerplate de las consultas cotidianas.

🤝 Se lleva bien con otros

Lee SQLite, PostgreSQL y MySQL mediante extensiones; exporta Parquet/CSV/JSON — el pegamento ideal para pipelines de datos.

🏆 ¿Por qué instalarlo con apt (y no un snap o una descarga manual)?

  • Las extensiones simplemente funcionan: este paquete usa la compilación glibc oficial de upstream, así que INSTALL httpfs; coincide con el repositorio oficial de extensiones linux_amd64/linux_arm64
  • Todas las versiones, incluidas las LTS: los archivos oficiales de Ubuntu no traen DuckDB — aquí desde Jammy hasta Resolute todas reciben la misma versión actual
  • Sin curl-to-bin: un repositorio firmado con GPG en lugar de descargar binarios a mano en /usr/local/bin
  • Actualizaciones automáticas: las nuevas versiones upstream llegan con tu apt upgrade habitual
  • Multi-arch: amd64 y arm64 — las arquitecturas para las que upstream publica binarios de la CLI para Linux
