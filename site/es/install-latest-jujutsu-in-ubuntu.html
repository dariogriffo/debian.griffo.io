¿Qué es Jujutsu?

Jujutsu es un sistema de control de versiones compatible con Git que ofrece un modelo más simple y potente que Git, manteniendo la compatibilidad con los repositorios Git existentes. Utiliza el comando jj, trata la copia de trabajo como un commit, ofrece un manejo de conflictos de primera clase y deshacer para cada operación — permitiéndote seguir usando alojamiento Git como GitHub o Forgejo.

🚀 Por qué importan las últimas versiones: Jujutsu evoluciona rápidamente con nuevos comandos, refinamientos de flujo de trabajo y mejoras de rendimiento. Usar la última versión significa acceso a las funciones más recientes y a las correcciones de interoperabilidad con Git.

⚡ Características clave de Jujutsu

🔄 Compatible con Git

Funciona directamente con repositorios Git existentes y remotos como GitHub y Forgejo.

↩️ Deshacer universal

Cada operación queda registrada y puede deshacerse con un solo comando.

🧩 Conflictos de primera clase

Los conflictos se registran en los commits y pueden resolverse en cualquier momento, no solo durante las fusiones.

📄 Copia de trabajo como commit

La copia de trabajo es en sí misma un commit, lo que elimina el área de preparación y simplifica los flujos de trabajo.

🎯 ¿Por qué Jujutsu?

  • Modelo más simple: Sin área de preparación y con un flujo de trabajo consistente y predecible
  • Experimentación segura: Deshaz cualquier operación, incluyendo rebases y fusiones
  • Interoperabilidad con Git: Sigue usando alojamiento Git y colabora con usuarios de Git
  • Edición de historial potente: Rebase sin esfuerzo y reescritura de commits

🆕 Destacados

  • El comando jj: Una CLI moderna sobre repositorios Git
  • Registro de operaciones: Historial completo de las operaciones del repositorio con deshacer
  • Manejo de conflictos: Los conflictos se almacenan en los commits, resolubles en cualquier momento
  • Backend Git: Interopera con remotos Git existentes

❓ Preguntas frecuentes

¿Está Jujutsu en los repositorios oficiales de Ubuntu?

No — Jujutsu no está empaquetado en los archivos oficiales de Ubuntu. Este repositorio es actualmente la única fuente apt y ofrece Jujutsu 0.43.0.

¿Cómo instalo el último Jujutsu en Ubuntu?

Añade el repositorio deb.griffo.io una sola vez con las instrucciones de arriba y ejecuta: sudo apt install jujutsu. Las nuevas versiones llegan con el sudo apt upgrade habitual.

¿Los paquetes están firmados y cómo se compilan?

Cada paquete está firmado con la clave GPG del repositorio (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) y se compila a partir de versiones upstream en repositorios públicos de GitHub que cualquiera puede inspeccionar.
