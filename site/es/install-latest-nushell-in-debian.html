¿Qué es Nushell?

Nushell (el comando nu) es una shell moderna construida alrededor de datos estructurados. En lugar de pasar texto plano entre comandos, cada pipeline transporta tablas y registros tipados — ls devuelve filas que puedes filtrar, ordenar y sumar, y JSON, YAML, CSV, TOML y SQLite son ciudadanos de primera clase. Combina un lenguaje de programación de verdad con una shell interactiva, dándote mensajes de error claros, autocompletado y resaltado de sintaxis desde el primer momento.

⚠️ Lista para usarse como shell de inicio de sesión: este paquete registra nu en /etc/shells al instalarse (y lo elimina limpiamente al desinstalarse), así que chsh -s /usr/bin/nu funciona de inmediato — sin ediciones manuales. El binario está enlazado estáticamente (musl), así que tu shell sigue funcionando sin importar las actualizaciones de librerías.
🚀 Siempre por delante de los archivos oficiales: Debian y Ubuntu congelan las versiones de los paquetes al publicar cada versión — este repositorio publica las nuevas versiones upstream normalmente en cuestión de horas. Un simple apt upgrade te mantiene siempre en la última versión.

⚡ Características principales de Nushell

📊 Pipelines estructurados

Los comandos emiten tablas, no texto: ls | where size > 1mb | sort-by modified — sin malabares con awk, cut o sed.

🔄 Todos los formatos de datos

Abre y transforma JSON, YAML, CSV, TOML, XML y SQLite de forma nativa: open Cargo.toml | get package.version.

🧠 Un lenguaje de verdad

Variables tipadas, closures, comandos personalizados y módulos — con mensajes de error de calidad de compilador que señalan el punto exacto.

🔌 Plugins incluidos

Incluye los plugins oficiales: polars (dataframes), query (JSON/XML/web scraping), formats, gstat e inc.

🔐 Lista como shell de inicio de sesión

Registrada en /etc/shells por el paquete — cambia con chsh -s /usr/bin/nu en cuanto apt termina.

📦 Cero dependencias

Un único binario estático musl. Sin runtime, sin librerías que se rompan — ideal para un programa en el que inicias sesión.

🏆 ¿Por qué instalarlo con apt?

  • Seguro para una shell: compilaciones enlazadas estáticamente, registradas y eliminadas de /etc/shells por los scripts del paquete — los detalles que importan cuando es tu shell de inicio de sesión
  • Plugins incluidos: el conjunto oficial de plugins viene en el mismo paquete, ajustado a la versión exacta de nu (los plugins lo requieren)
  • Sin curl-to-bin: un repositorio firmado con GPG en lugar de descargar binarios a mano en /usr/local/bin
  • Actualizaciones automáticas: las nuevas versiones upstream llegan con tu apt upgrade habitual — importante, porque Nushell publica una versión cada mes
  • Multi-arch: amd64, arm64, armhf y riscv64

No — Nushell no está empaquetado en los archivos oficiales de Debian. Este repositorio es actualmente la única fuente apt y ofrece Nushell 0.114.1.

¿Cómo instalo el último Nushell en Debian?

Añade el repositorio deb.griffo.io una sola vez con las instrucciones de arriba y ejecuta: sudo apt install nushell. Las nuevas versiones llegan con el sudo apt upgrade habitual.

¿Los paquetes están firmados y cómo se compilan?

Cada paquete está firmado con la clave GPG del repositorio (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) y se compila a partir de versiones upstream en repositorios públicos de GitHub que cualquiera puede inspeccionar.
