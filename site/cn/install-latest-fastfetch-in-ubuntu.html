什么是 Fastfetch？

Fastfetch 是一款类 neofetch 的系统信息获取工具，能在终端中将系统信息与您的发行版 logo 一同精美呈现——操作系统、内核、硬件、运行时间、软件包、桌面环境等等。它主要使用 C 编写，注重性能与可定制性；在 neofetch 归档停止维护后，它已成为事实上的继任者，并且开发活跃、发布频繁。

😤 官方源版本的问题：较旧的 Ubuntu 版本根本不提供 fastfetch；即便官方源中有它，版本也被冻结在导入时的状态。上游进展很快：新 logo、新检测模块以及针对新硬件的修复不断合入，官方源版本会悄然落后。本仓库提供紧跟上游的最新构建，同时保持与官方源软件包完全相同的文件布局，双向切换都无缝衔接。
🚀 始终领先于官方软件源：Debian 和 Ubuntu 在发布时会冻结软件包版本——本仓库通常在上游发布后数小时内即发布新版本。只需 apt upgrade 即可始终保持最新版本。

⚡ Fastfetch 的主要特性

🏎️ 为速度而生

使用 C 编写并采用并发检测——毫秒级渲染完整的系统信息，启动时间仅为 neofetch 的一小部分。

🧩 50+ 信息模块

操作系统、内核、CPU、GPU、内存、磁盘、电池、显示器、DE/WM、终端、软件包、媒体播放器等等——每一项均可单独配置。

🎨 内置预设

本软件包将所有内置预设安装到 /usr/share/fastfetch/presets/——包括高度还原的 neofetch 预设，让您即刻找到熟悉的感觉。

📝 JSONC 配置

一个文档完善的 JSONC 配置文件即可控制模块、格式和 logo。使用 fastfetch --gen-config 生成起始配置。

🖼️ 灵活的 Logo

数百个内置发行版 logo，还支持通过 sixel/kitty/iTerm 协议显示图片 logo、自定义 ASCII 艺术，或完全不显示。

🐚 Shell 补全

提供 bash、zsh 和 fish 补全以及 man 手册页——安装到标准 Debian 路径，与官方源软件包保持一致。

🏆 为什么选择本仓库而不是官方源版本？

  • 版本最新：上游新版本数小时内即可获得，而不是在整个 Debian 发行周期内被冻结
  • 布局一致：与 Debian 官方软件包逐文件对齐——预设、补全、man 手册页都在标准位置
  • 无缝切换：从本仓库安装只是升级官方源软件包；移除仓库后可干净回退
  • 自动更新：新版本随您日常的 apt upgrade 一同到达
  • 多架构支持：amd64、arm64、armhf、ppc64el、s390x 和 riscv64
