什么是 yq？

yq 是一款轻量、便携的结构化数据命令行处理器——支持 YAML、JSON、XML、CSV、TOML 和 properties 文件，由 Mike Farah 使用 Go 编写。它采用类 jq 语法，您在 jq 上学到的一切都可以直接迁移：在脚本和流水线中查询、转换、合并以及原地编辑文件，只需一个零依赖的二进制文件。

⚠️ 这是哪个 yq？这是 mikefarah/yq（v4.x，Go）——绝大多数 CI 示例和文档中使用的那一个。它不是某些官方源中基于 python 的 yq jq 封装器；本软件包可干净地取代后者，且不需要 python 或 jq。
🚀 始终领先于官方软件源：Debian 和 Ubuntu 在发布时会冻结软件包版本——本仓库通常在上游发布后数小时内即发布新版本。只需 apt upgrade 即可始终保持最新版本。

⚡ yq 的主要特性

🔄 六种格式

YAML、JSON、XML、CSV/TSV、TOML 和 properties——读取一种、输出另一种：yq -o=json '.' config.yaml 一步完成转换。

🧠 类 jq 语法

熟悉的路径、管道、函数和运算符。只要会写 jq 过滤器，就已经会在 YAML 上使用 yq。

✏️ 原地编辑

yq -i '.spec.replicas = 3' deploy.yaml——以脚本方式编辑 Kubernetes 清单、CI 配置和 Compose 文件，注释完好保留。

📦 零依赖

单个静态 Go 二进制文件。没有运行时、没有库、没有 python——是精简容器和 CI runner 的理想选择。

🔀 合并与锚点

深度合并多个文件、解析 YAML 锚点和别名、处理多文档流——把 YAML 的疑难场景都处理得当。

📖 附带 man 手册页

本软件包在二进制文件之外还提供完整的 man 手册页——man yq 可离线使用，符合 Debian 规范。

🏆 为什么用 apt 安装？

  • 适合大规模部署：在基础镜像中固定一个仓库，每台服务器、每个容器和 CI runner 都能获得同样新版的 yq
  • 告别 curl 下载二进制：使用 GPG 签名的仓库，而不是手动把二进制文件下载到 /usr/local/bin
  • 正确的 yq：在所有机器上保证 mikefarah/yq v4 语义——不会因某些机器上的 python 封装器而遭遇意外
  • 自动更新：上游新版本随您日常的 apt upgrade 一同到达
  • 多架构支持：amd64、arm64、armhf、ppc64el、s390x 和 riscv64

❓ 常见问题

yq 在官方 Ubuntu 软件源中吗？

不在 — 官方 Ubuntu 源没有打包 yq。本仓库目前是唯一的 apt 来源，提供 yq 4.53.3。

如何在 Ubuntu 上安装最新版 yq？

按上方说明一次性添加 deb.griffo.io 仓库，然后运行：sudo apt install yq。之后通过常规的 sudo apt upgrade 获取新版本。

软件包是否有签名？如何构建？

每个软件包都使用仓库的 GPG 密钥（EA0F721D231FDD3A0A17B9AC7808B4DD62C41256）签名，并基于上游发布在公开的 GitHub 仓库中构建，任何人都可以审查。
