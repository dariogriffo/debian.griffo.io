什么是 Jujutsu？

Jujutsu 是一款Git 兼容的版本控制系统，提供比 Git 更简单、更强大的模型，同时保持与现有 Git 仓库的向后兼容性。它使用 jj 命令，将工作副本本身视为一个提交，提供一流的冲突处理能力以及针对每个操作的撤销功能——让您底层依然可以继续使用 GitHub 或 Forgejo 等 Git 托管服务。

🚀 为什么最新版本很重要：Jujutsu 正在快速发展，不断推出新命令、工作流优化和性能改进。使用最新版本意味着能够第一时间获得最新功能和 Git 互操作性修复。

⚡ Jujutsu 的核心功能

🔄 兼容 Git

可直接配合现有 Git 仓库以及 GitHub、Forgejo 等远程仓库使用。

↩️ 通用撤销

每个操作都会被记录，只需一条命令即可撤销。

🧩 一流的冲突处理

冲突会被记录在提交中，可以随时解决，而不仅限于合并期间。

📄 工作副本即提交

工作副本本身就是一个提交，省去了暂存区，简化了工作流程。

🎯 为什么选择 Jujutsu？

  • 更简单的模型：没有暂存区，工作流程一致且可预测
  • 安全的实验：可撤销任何操作，包括 rebase 和合并
  • Git 互操作：继续使用 Git 托管服务，与 Git 用户协作
  • 强大的历史编辑：轻松完成 rebase 和提交重写

🆕 亮点功能

  • jj 命令：面向 Git 仓库的现代化命令行工具
  • 操作日志：完整记录仓库操作历史，支持撤销
  • 冲突处理：冲突存储在提交中，可随时解决
  • Git 后端：与现有 Git 远程仓库互操作
