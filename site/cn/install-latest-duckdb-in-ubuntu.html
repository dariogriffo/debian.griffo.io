什么是 DuckDB？

DuckDB 是一款快速的进程内分析型 SQL 数据库——可以把它看作“分析领域的 SQLite”。它在调用方进程内运行，无需搭建任何服务器，可直接查询 CSV、Parquet 和 JSON 文件（SELECT * FROM 'data.parquet'），并提供丰富、友好的 SQL 方言，在分析型工作负载上性能出色。本软件包提供官方 duckdb CLI。

⚠️ 无服务器、无守护进程：DuckDB 是嵌入式数据库——它在您的进程（或 CLI）内运行，因此没有需要配置的服务、没有需要开放的端口，也没有 systemd 单元。安装后直接运行 duckdb 即可。同样因为这个原因，一个数据库文件同一时间只属于一个进程。
🚀 始终领先于官方软件源：Debian 和 Ubuntu 在发布时会冻结软件包版本——本仓库通常在上游发布后数小时内即发布新版本，覆盖包括各 LTS 在内的所有受支持发行版。只需 apt upgrade 即可始终保持最新版本。

⚡ DuckDB 的主要特性

📊 为分析而生

列式、向量化引擎每秒处理数百万行——以 OLAP 速度完成聚合、连接和窗口函数。

📁 直接查询文件

SELECT * FROM 'events.parquet'——CSV、Parquet 和 JSON 无需任何导入步骤即可查询（和写入）。

🪶 零配置

像 SQLite 一样运行在进程内：一个二进制文件、一个可选的数据库文件，无需任何管理。

🔌 运行时扩展

INSTALL httpfs; 从官方仓库拉取签名扩展——S3/HTTP 访问、空间数据、全文搜索等等。

🗣️ 友好的 SQL

SELECT * EXCLUDE、GROUP BY ALL、list/struct 类型、PIVOT——一种为日常查询去除样板代码的现代方言。

🤝 与其他系统协作

通过扩展读取 SQLite、PostgreSQL 和 MySQL；导出 Parquet/CSV/JSON——是数据流水线的理想粘合剂。

🏆 为什么用 apt 安装（而不是 snap 或手动下载）？

  • 扩展开箱即用：本软件包使用上游官方 glibc 构建，因此 INSTALL httpfs; 与官方 linux_amd64/linux_arm64 扩展仓库完全匹配
  • 覆盖包括各 LTS 在内的每个发行版：官方 Ubuntu 软件源不提供 DuckDB——在这里从 Jammy 到 Resolute 都能获得同样的最新版本
  • 告别 curl 下载二进制：使用 GPG 签名的仓库，而不是手动把二进制文件下载到 /usr/local/bin
  • 自动更新：上游新版本随您日常的 apt upgrade 一同到达
  • 多架构支持：amd64 和 arm64——即上游发布 Linux CLI 二进制文件的架构

❓ 常见问题

DuckDB 在官方 Ubuntu 软件源中吗？

不在 — 官方 Ubuntu 源没有打包 DuckDB。本仓库目前是唯一的 apt 来源，提供 DuckDB 1.5.5。

如何在 Ubuntu 上安装最新版 DuckDB？

按上方说明一次性添加 deb.griffo.io 仓库，然后运行：sudo apt install duckdb。之后通过常规的 sudo apt upgrade 获取新版本。

软件包是否有签名？如何构建？

每个软件包都使用仓库的 GPG 密钥（EA0F721D231FDD3A0A17B9AC7808B4DD62C41256）签名，并基于上游发布在公开的 GitHub 仓库中构建，任何人都可以审查。

支持哪些 Ubuntu 版本？

Ubuntu 22.04 Jammy、24.04 Noble、25.10 Questing 和 26.04 Resolute。
