什么是 Nushell？

Nushell（即 nu 命令）是一款围绕结构化数据构建的现代 shell。命令之间传递的不再是纯文本，而是带类型的表格和记录——ls 返回的行可以直接过滤、排序和求和，JSON、YAML、CSV、TOML 和 SQLite 都是一等公民。它将一门真正的编程语言与交互式 shell 结合在一起，开箱即用地提供清晰的错误信息、自动补全和语法高亮。

⚠️ 可直接用作登录 shell：本软件包在安装时会把 nu 注册到 /etc/shells（卸载时干净移除），因此 chsh -s /usr/bin/nu 立即可用——无需手动编辑。二进制文件为静态链接（musl），无论系统库如何升级，您的 shell 都能正常工作。
🚀 始终领先于官方软件源：Debian 和 Ubuntu 在发布时会冻结软件包版本——本仓库通常在上游发布后数小时内即发布新版本。只需 apt upgrade 即可始终保持最新版本。

⚡ Nushell 的主要特性

📊 结构化管道

命令输出的是表格而非文本：ls | where size > 1mb | sort-by modified——无需 awk、cut 或 sed 的各种技巧。

🔄 支持各种数据格式

原生打开并转换 JSON、YAML、CSV、TOML、XML 和 SQLite：open Cargo.toml | get package.version。

🧠 真正的编程语言

带类型的变量、闭包、自定义命令和模块——错误信息达到编译级质量，直接指向出错的确切位置。

🔌 内置插件

附带官方插件：polars（dataframe）、query（JSON/XML/网页抓取）、formats、gstat 和 inc。

🔐 登录 shell 就绪

软件包自动注册到 /etc/shells——apt 安装完成的那一刻即可用 chsh -s /usr/bin/nu 切换。

📦 零依赖

单个静态 musl 二进制文件。没有运行时、没有会损坏的库——对于您每天登录使用的程序来说再理想不过。

🏆 为什么用 apt 安装？

  • 作为 shell 足够安全：静态链接构建，由软件包脚本负责在 /etc/shells 中注册和注销——当它是您的登录 shell 时，这些细节至关重要
  • 内置插件：官方插件集随同一软件包一起提供，与 nu 的确切版本相匹配（插件对此有要求）
  • 告别 curl 下载二进制：使用 GPG 签名的仓库，而不是手动把二进制文件下载到 /usr/local/bin
  • 自动更新：上游新版本随您日常的 apt upgrade 一同到达——这很重要，因为 Nushell 每月发布新版
  • 多架构支持：amd64、arm64、armhf 和 riscv64

❓ 常见问题

Nushell 在官方 Debian 软件源中吗？

不在 — 官方 Debian 源没有打包 Nushell。本仓库目前是唯一的 apt 来源，提供 Nushell 0.114.1。

如何在 Debian 上安装最新版 Nushell？

按上方说明一次性添加 deb.griffo.io 仓库，然后运行：sudo apt install nushell。之后通过常规的 sudo apt upgrade 获取新版本。

软件包是否有签名？如何构建？

每个软件包都使用仓库的 GPG 密钥（EA0F721D231FDD3A0A17B9AC7808B4DD62C41256）签名，并基于上游发布在公开的 GitHub 仓库中构建，任何人都可以审查。

支持哪些 Debian 版本？

Debian 12 Bookworm、Debian 13 Trixie、Forky 和 Sid。
