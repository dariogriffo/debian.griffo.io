O que é o uncloud?

uncloud é uma ferramenta leve de clustering e orquestração de containers criada por Pasha Sviderski que permite implantar e gerenciar aplicações web em VMs na nuvem e servidores bare metal com sobrecarga mínima de gerenciamento de cluster. Ele preenche a lacuna entre a simplicidade do Docker e o poder do Kubernetes, oferecendo a simplicidade do Docker com o poder multi-máquina.

🎯 O Ponto de Equilíbrio: Plataformas modernas na nuvem como o Heroku oferecem experiências incríveis para desenvolvedores, mas a preços premium. O Kubernetes oferece poder, mas exige conhecimento especializado significativo. O uncloud encontra o meio-termo pragmático - possua sua infraestrutura, mantenha a simplicidade ao crescer, construa com primitivas comprovadas.
⚠️ Status de Desenvolvimento: uncloud está atualmente em desenvolvimento ativo e não está pronto para uso em produção. Os recursos podem mudar significativamente entre versões. Perfeito para experimentação e aprendizado!
🚀 Por que as últimas versões são importantes: o uncloud está evoluindo rapidamente com lançamentos frequentes contendo novos recursos de orquestração, melhorias de desempenho e capacidades multi-máquina aprimoradas. As versões mais recentes incluem melhor rede mesh WireGuard, descoberta de serviços aprimorada e compatibilidade aprimorada com Docker Compose.

⚡ Principais Recursos do uncloud

🌐 Implantação Multi-Máquina

Implante em VMs na nuvem, servidores dedicados e bare metal. Combine diferentes provedores em um ambiente de computação unificado.

🐳 Compatível com Docker Compose

Use o formato familiar do Docker Compose para definir serviços. Sem necessidade de aprender novas linguagens de configuração ou YAML complexo.

🔒 Rede Privada sem Configuração

Mesh WireGuard automático com descoberta de peers e travessia de NAT. Os containers recebem IPs únicos para comunicação direta entre máquinas.

🎯 Sem Plano de Controle

Design totalmente descentralizado elimina pontos únicos de falha. Cada máquina mantém o estado sincronizado do cluster por meio de comunicação peer-to-peer.

🔍 Descoberta de Serviços

Servidor DNS integrado resolve nomes de serviços para IPs de containers. Balanceamento de carga automático e ingress com HTTPS via Caddy.

📦 Armazenamento Persistente

Execute serviços com estado usando volumes Docker gerenciados entre máquinas. Persistência de dados sem orquestração de armazenamento complexa.

🎯 Exemplos de uso básico

Inicialize seu primeiro cluster:

# Initialize cluster on first machine uc machine init root@your-server-ip # Add more machines to cluster uc machine add --name server2 root@second-server-ip # List cluster machines uc machine ls

Implante aplicações:

# Deploy from Docker image with HTTPS uc run -p app.example.com:8000/https image/my-app # Deploy from Docker Compose file uc compose up -f compose.yaml # Scale service across machines uc scale my-app 3 # List running services uc ls

Gerencie implantações:

# Update service with new image uc update my-app image/my-app:v2 # View service logs uc logs my-app # Remove service uc rm my-app # Clean up cluster uc machine rm server2

❓ Perguntas frequentes

O Uncloud está nos repositórios oficiais do Ubuntu?

Não — Uncloud não está empacotado nos arquivos oficiais do Ubuntu. Este repositório é atualmente a única fonte apt, oferecendo Uncloud 0.20.0.

Como instalo o Uncloud mais recente no Ubuntu?

Adicione o repositório deb.griffo.io uma única vez com as instruções acima e execute: sudo apt install uncloud. Novos lançamentos chegam pelo sudo apt upgrade normal.

Os pacotes são assinados e como são construídos?

Cada pacote é assinado com a chave GPG do repositório (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) e construído a partir de lançamentos upstream em repositórios públicos do GitHub que qualquer pessoa pode inspecionar.
