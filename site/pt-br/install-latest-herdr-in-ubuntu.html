O que é o herdr?

herdr é um multiplexador de agentes que vive no seu terminal — um gerenciador de espaço de trabalho para executar e orquestrar vários agentes de codificação com IA lado a lado. Escrito em Rust e distribuído como um único binário estático, ele permite conduzir toda uma frota de agentes (Claude Code, Codex, Gemini, Aider e mais) a partir de uma única TUI amigável ao teclado, com sessões persistentes, git worktrees e acesso remoto via SSH.

🤖 Feito para agentes de codificação com IA:
  • Execute vários agentes de uma vez — cada um em seu próprio painel, aba e espaço de trabalho
  • Sessões persistentes que sobrevivem a desconexões, como um multiplexador para agentes
  • Ciente de git worktrees — isole cada agente em seu próprio branch
  • Remoto via SSH — conecte-se a um servidor herdr rodando em qualquer lugar
🚀 Por que as últimas versões são importantes: o herdr evolui rapidamente, com canais estáveis e de pré-visualização e lançamentos frequentes que adicionam novas integrações de agentes, recursos de espaço de trabalho e melhorias na TUI. Instalar a partir do deb.griffo.io mantém você sempre na versão mais recente sem precisar executar o autoatualizador integrado.

⚡ Principais Recursos do herdr

🤖 Vários Agentes Lado a Lado

Inicie e orquestre vários agentes de codificação com IA ao mesmo tempo, cada um em seu próprio painel e aba, e alterne entre eles sem sair do terminal.

🗂️ Espaços de Trabalho e Sessões

Agrupe seu trabalho em sessões e espaços de trabalho nomeados e persistentes que continuam em execução em segundo plano e se reconectam instantaneamente.

🌿 Git Worktrees

Dê a cada agente seu próprio git worktree e branch, para que o trabalho paralelo nunca colida e os diffs permaneçam limpos e revisáveis.

🖥️ Remoto via SSH

Conecte-se via SSH a um servidor herdr rodando em qualquer máquina e conduza sessões remotas como se fossem locais.

⌨️ TUI Controlada por Teclado

Uma interface de terminal rápida, com mouse opcional, com atalhos de teclado, abas e painéis configuráveis — tudo a um toque de distância.

🔌 Integrações e Extensível

Integrações de agentes integradas, além de uma API de socket e arquivo de configuração, com um ecossistema crescente de plugins de primeira classe ao seu redor.

🎯 Exemplos de uso básico

Iniciar e conectar à sua sessão:

# Launch or attach to the persistent session herdr # Use or create a named session herdr --session work # Attach to a named session herdr session attach work

Trabalhar com servidores remotos:

# Attach through SSH to a remote herdr server herdr --remote user@server # Open a named session on a remote host herdr --remote user@server --session build

# Workspace, worktree and agent helpers over the socket API herdr workspace list herdr worktree list herdr agent list # Show local client and running server status herdr status

❓ Perguntas frequentes

O herdr está nos repositórios oficiais do Ubuntu?

Não — herdr não está empacotado nos arquivos oficiais do Ubuntu. Este repositório é atualmente a única fonte apt, oferecendo herdr 0.7.5.

Como instalo o herdr mais recente no Ubuntu?

Adicione o repositório deb.griffo.io uma única vez com as instruções acima e execute: sudo apt install herdr. Novos lançamentos chegam pelo sudo apt upgrade normal.

Os pacotes são assinados e como são construídos?

Cada pacote é assinado com a chave GPG do repositório (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) e construído a partir de lançamentos upstream em repositórios públicos do GitHub que qualquer pessoa pode inspecionar.
