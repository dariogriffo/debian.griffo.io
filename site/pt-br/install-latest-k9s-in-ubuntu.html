O que é o k9s?

O k9s é uma interface de terminal para gerenciar clusters Kubernetes. Ele oferece uma interface rápida, orientada a teclado, para observar e interagir com seus clusters em execução em tempo real — navegando por recursos, visualizando logs, executando comandos dentro de pods e aplicando mudanças — sem precisar digitar constantemente longos comandos kubectl.

🚀 Por que as últimas versões são importantes: O k9s lança atualizações frequentes com novas visualizações, plugins e suporte à API do Kubernetes. Mantê-lo atualizado garante compatibilidade com as versões e recursos mais recentes dos clusters.

⚡ Principais Recursos do k9s

⌨️ Orientado a Teclado

Navegue e gerencie recursos do cluster rapidamente com atalhos intuitivos.

📜 Logs ao Vivo & Exec

Transmita logs de pods e abra um shell dentro de containers diretamente pela interface.

🔍 Visualizações de Recursos

Navegue por deployments, pods, services e mais, com status em tempo real.

🧩 Extensível

Skins, aliases e plugins personalizados para adaptar seu fluxo de trabalho.

🎯 Por que o k9s?

  • Mais rápido que o kubectl: Observe e aja sobre recursos sem comandos longos
  • Tempo real: Visão ao vivo do estado do cluster e da saúde dos recursos
  • Navegação poderosa: Salte entre recursos relacionados instantaneamente
  • Customizável: Skins, atalhos e plugins que se adaptam à sua equipe

🆕 Destaques

  • Navegação do Cluster: Explore todos os tipos de recursos do Kubernetes
  • Logs & Shell: Acompanhe logs e execute comandos dentro de pods
  • Ações em Recursos: Escale, exclua e edite direto pela interface
  • Plugins: Estenda com comandos personalizados

❓ Perguntas frequentes

O k9s está nos repositórios oficiais do Ubuntu?

Não — k9s não está empacotado nos arquivos oficiais do Ubuntu. Este repositório é atualmente a única fonte apt, oferecendo k9s 0.51.0.

Como instalo o k9s mais recente no Ubuntu?

Adicione o repositório deb.griffo.io uma única vez com as instruções acima e execute: sudo apt install k9s. Novos lançamentos chegam pelo sudo apt upgrade normal.

Os pacotes são assinados e como são construídos?

Cada pacote é assinado com a chave GPG do repositório (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) e construído a partir de lançamentos upstream em repositórios públicos do GitHub que qualquer pessoa pode inspecionar.
