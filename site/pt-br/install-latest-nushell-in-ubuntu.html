O que é o Nushell?

O Nushell (o comando nu) é um shell moderno construído em torno de dados estruturados. Em vez de passar texto puro entre comandos, cada pipeline carrega tabelas e registros tipados — ls retorna linhas que você pode filtrar, ordenar e somar, e JSON, YAML, CSV, TOML e SQLite são cidadãos de primeira classe. Ele combina uma linguagem de programação de verdade com um shell interativo, oferecendo mensagens de erro claras, autocompletar e realce de sintaxe prontos para uso.

⚠️ Pronto para usar como shell de login: este pacote registra o nu em /etc/shells na instalação (e o remove de forma limpa na desinstalação), então chsh -s /usr/bin/nu funciona imediatamente — sem edição manual. O binário é vinculado estaticamente (musl), então seu shell continua funcionando independentemente de atualizações de bibliotecas.
🚀 Sempre à frente dos repositórios oficiais: Debian e Ubuntu congelam as versões dos pacotes a cada lançamento — este repositório publica novas versões upstream normalmente em poucas horas. Um simples apt upgrade mantém você sempre na versão mais recente.

⚡ Principais Recursos do Nushell

📊 Pipelines Estruturados

Comandos emitem tabelas, não texto: ls | where size > 1mb | sort-by modified — sem ginástica de awk, cut ou sed.

🔄 Todos os Formatos de Dados

Abra e transforme JSON, YAML, CSV, TOML, XML e SQLite nativamente: open Cargo.toml | get package.version.

🧠 Uma Linguagem de Verdade

Variáveis tipadas, closures, comandos personalizados e módulos — com mensagens de erro de qualidade de compilador que apontam o trecho exato.

🔌 Plugins Incluídos

Traz os plugins oficiais: polars (dataframes), query (JSON/XML/web scraping), formats, gstat e inc.

🔐 Pronto para Shell de Login

Registrado em /etc/shells pelo pacote — troque com chsh -s /usr/bin/nu assim que o apt terminar.

📦 Zero Dependências

Um único binário estático em musl. Sem runtime, sem bibliotecas para quebrar — ideal para um programa no qual você faz login.

🏆 Por que instalar com o apt (e não um snap ou download manual)?

  • Seguro para um shell: builds vinculados estaticamente, registrado e desregistrado em /etc/shells pelos scripts do pacote — os detalhes que importam quando é o seu shell de login
  • Plugins incluídos: o conjunto oficial de plugins vem no mesmo pacote, casado com a versão exata do nu (os plugins exigem isso)
  • Sem curl-para-bin: um repositório assinado com GPG em vez de baixar binários manualmente para /usr/local/bin
  • Atualizações automáticas: novos lançamentos upstream chegam com o seu apt upgrade de sempre — importante, já que o Nushell lança mensalmente
  • Multi-arquitetura: amd64, arm64, armhf e riscv64

❓ Perguntas frequentes

O Nushell está nos repositórios oficiais do Ubuntu?

Não — Nushell não está empacotado nos arquivos oficiais do Ubuntu. Este repositório é atualmente a única fonte apt, oferecendo Nushell 0.114.1.

Como instalo o Nushell mais recente no Ubuntu?

Adicione o repositório deb.griffo.io uma única vez com as instruções acima e execute: sudo apt install nushell. Novos lançamentos chegam pelo sudo apt upgrade normal.

Os pacotes são assinados e como são construídos?

Cada pacote é assinado com a chave GPG do repositório (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) e construído a partir de lançamentos upstream em repositórios públicos do GitHub que qualquer pessoa pode inspecionar.
