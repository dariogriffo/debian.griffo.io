🥋 Установка последней версии Jujutsu в Debian

Git-совместимая система контроля версий

Последняя версия: 0.43.0 · обновлено 2026-07-16
← На главную

Что такое Jujutsu?

Jujutsu — это Git-совместимая система контроля версий, которая предлагает более простую и мощную модель, чем Git, оставаясь при этом обратно совместимой с существующими репозиториями Git. Она использует команду jj, рассматривает рабочую копию как коммит, обеспечивает полноценную обработку конфликтов и отмену для каждой операции — позволяя вам продолжать использовать Git-хостинги вроде GitHub или Forgejo под капотом.

🚀 Почему важны последние версии: Jujutsu быстро развивается — с новыми командами, доработками рабочего процесса и улучшениями производительности. Использование последней версии означает доступ к новейшим функциям и исправлениям взаимодействия с Git.

⚡ Ключевые возможности Jujutsu

🔄 Совместимость с Git

Работает напрямую с существующими репозиториями и удалёнными репозиториями Git, такими как GitHub и Forgejo.

↩️ Универсальная отмена

Каждая операция записывается и может быть отменена одной командой.

🧩 Полноценные конфликты

Конфликты фиксируются в коммитах и могут быть разрешены в любой момент, а не только во время слияний.

📄 Рабочая копия как коммит

Рабочая копия сама является коммитом, что убирает область подготовки и упрощает рабочие процессы.

🎯 Почему Jujutsu?

  • Более простая модель: Отсутствие области подготовки и последовательный, предсказуемый рабочий процесс
  • Безопасные эксперименты: Отмена любой операции, включая перебазирование и слияния
  • Совместимость с Git: Продолжайте использовать Git-хостинги и сотрудничать с пользователями Git
  • Мощное редактирование истории: Лёгкое перебазирование и переписывание коммитов

🆕 Основные возможности

  • Команда jj: Современный CLI поверх репозиториев Git
  • Журнал операций: Полная история операций репозитория с возможностью отмены
  • Обработка конфликтов: Конфликты хранятся в коммитах, разрешимы в любой момент
  • Движок Git: Взаимодействует с существующими удалёнными репозиториями Git
