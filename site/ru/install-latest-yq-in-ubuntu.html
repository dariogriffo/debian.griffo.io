Что такое yq?

yq — это лёгкий портативный консольный процессор структурированных данных — YAML, JSON, XML, CSV, TOML и properties-файлов, — написанный на Go Майком Фарой. Он использует синтаксис в стиле jq, поэтому всё, что вы знаете из jq, переносится напрямую: запрашивайте, преобразуйте, объединяйте и редактируйте файлы на месте, в скриптах и пайплайнах, с одним бинарным файлом без зависимостей.

⚠️ Какой это yq? Это mikefarah/yq (v4.x, Go) — тот самый, что используется в большинстве примеров CI и документации. Это не основанная на python обёртка над jq под именем yq, встречающаяся в некоторых архивах; этот пакет чисто её замещает, и ни python, ни jq не требуются.
🚀 Всегда впереди официальных архивов: Debian и Ubuntu замораживают версии пакетов при выпуске релиза — этот репозиторий публикует новые версии обычно в течение нескольких часов после релиза upstream. Простой apt upgrade всегда держит вас на последней версии.

⚡ Ключевые возможности yq

🔄 Шесть форматов

YAML, JSON, XML, CSV/TSV, TOML и properties — читайте один, выводите другой: yq -o=json '.' config.yaml преобразует за один шаг.

🧠 Синтаксис в стиле jq

Знакомые пути, пайпы, функции и операторы. Если вы умеете писать фильтр jq, вы уже умеете использовать yq для YAML.

✏️ Редактирование на месте

yq -i '.spec.replicas = 3' deploy.yaml — скриптовые правки манифестов Kubernetes, конфигураций CI и файлов Compose с сохранением комментариев.

📦 Ноль зависимостей

Один статический бинарный файл на Go. Ни рантайма, ни библиотек, ни python — идеально для минимальных контейнеров и CI-раннеров.

🔀 Слияние и якоря

Глубокое слияние нескольких файлов, разрешение якорей и алиасов YAML, обработка многодокументных потоков — сложные случаи YAML сделаны правильно.

📖 Man-страница в комплекте

Этот пакет поставляет полную man-страницу вместе с бинарным файлом — man yq работает офлайн, в соответствии с конвенциями Debian.

🏆 Зачем устанавливать через apt?

  • Удобно для парка машин: закрепите один репозиторий в базовых образах — и каждый сервер, контейнер и CI-раннер получит один и тот же актуальный yq
  • Без curl-to-bin: репозиторий с GPG-подписью вместо ручной загрузки бинарных файлов в /usr/local/bin
  • Правильный yq: гарантирует семантику mikefarah/yq v4 повсюду — никаких сюрпризов от python-обёртки на отдельных машинах
  • Автоматические обновления: новые релизы upstream приходят с обычным apt upgrade
  • Мульти-архитектура: amd64, arm64, armhf, ppc64el, s390x и riscv64

❓ Часто задаваемые вопросы

Есть ли yq в официальных репозиториях Ubuntu?

Нет — yq не упакован в официальных архивах Ubuntu. Этот репозиторий сейчас единственный источник apt и предлагает yq 4.53.3.

Как установить последнюю версию yq в Ubuntu?

Добавьте репозиторий deb.griffo.io один раз по инструкции выше, затем выполните: sudo apt install yq. Новые версии приходят через обычный sudo apt upgrade.

Подписаны ли пакеты и как они собираются?

Каждый пакет подписан GPG-ключом репозитория (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) и собирается из релизов upstream в публичных репозиториях GitHub, которые может проверить любой.

Какие версии Ubuntu поддерживаются?

Ubuntu 22.04 Jammy, 24.04 Noble, 25.10 Questing и 26.04 Resolute.
