Что такое Nushell?

Nushell (команда nu) — это современная оболочка, построенная вокруг структурированных данных. Вместо передачи сырого текста между командами каждый пайплайн несёт типизированные таблицы и записи — ls возвращает строки, которые можно фильтровать, сортировать и суммировать, а JSON, YAML, CSV, TOML и SQLite поддерживаются как полноправные форматы. Nushell сочетает настоящий язык программирования с интерактивной оболочкой, из коробки давая понятные сообщения об ошибках, автодополнение и подсветку синтаксиса.

⚠️ Готов к работе как login-оболочка: этот пакет регистрирует nu в /etc/shells при установке (и аккуратно удаляет запись при удалении пакета), поэтому chsh -s /usr/bin/nu работает сразу — без ручного редактирования. Бинарный файл статически слинкован (musl), так что ваша оболочка продолжает работать независимо от обновлений библиотек.
🚀 Всегда впереди официальных архивов: Debian и Ubuntu замораживают версии пакетов при выпуске релиза — этот репозиторий публикует новые версии обычно в течение нескольких часов после релиза upstream. Простой apt upgrade всегда держит вас на последней версии.

⚡ Ключевые возможности Nushell

📊 Структурированные пайплайны

Команды выдают таблицы, а не текст: ls | where size > 1mb | sort-by modified — без гимнастики с awk, cut и sed.

🔄 Любой формат данных

Открывайте и преобразуйте JSON, YAML, CSV, TOML, XML и SQLite нативно: open Cargo.toml | get package.version.

🧠 Настоящий язык

Типизированные переменные, замыкания, собственные команды и модули — с сообщениями об ошибках уровня компилятора, указывающими на точное место.

🔌 Плагины в комплекте

Поставляет официальные плагины: polars (датафреймы), query (JSON/XML/веб-скрейпинг), formats, gstat и inc.

🔐 Готов быть login-оболочкой

Пакет регистрирует его в /etc/shells — переключайтесь командой chsh -s /usr/bin/nu, как только apt закончит установку.

📦 Ноль зависимостей

Один статический бинарный файл (musl). Ни рантайма, ни библиотек, которые могут сломаться — идеально для программы, через которую вы входите в систему.

🏆 Зачем устанавливать через apt?

  • Безопасно для оболочки: статически слинкованные сборки, регистрация и снятие регистрации в /etc/shells скриптами пакета — детали, которые важны, когда это ваша login-оболочка
  • Плагины в комплекте: официальный набор плагинов поставляется в том же пакете и соответствует точной версии nu (плагины этого требуют)
  • Без curl-to-bin: репозиторий с GPG-подписью вместо ручной загрузки бинарных файлов в /usr/local/bin
  • Автоматические обновления: новые релизы upstream приходят с обычным apt upgrade — важно, ведь Nushell выпускается ежемесячно
  • Мульти-архитектура: amd64, arm64, armhf и riscv64

❓ Часто задаваемые вопросы

Есть ли Nushell в официальных репозиториях Debian?

Нет — Nushell не упакован в официальных архивах Debian. Этот репозиторий сейчас единственный источник apt и предлагает Nushell 0.114.1.

Как установить последнюю версию Nushell в Debian?

Добавьте репозиторий deb.griffo.io один раз по инструкции выше, затем выполните: sudo apt install nushell. Новые версии приходят через обычный sudo apt upgrade.

Подписаны ли пакеты и как они собираются?

Каждый пакет подписан GPG-ключом репозитория (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) и собирается из релизов upstream в публичных репозиториях GitHub, которые может проверить любой.

Какие версии Debian поддерживаются?

Debian 12 Bookworm, Debian 13 Trixie, Forky и Sid.
