Что такое Neovim?

Neovim — это гипер-расширяемый текстовый редактор на основе Vim, который переосмысливает классическую кодовую базу Vim для современной разработки. Он поставляется со встроенным клиентом Language Server Protocol, синтаксисом и пониманием кода на основе Treesitter, асинхронным движком задач и первоклассными сценариями на Lua — поэтому плагины и конфигурация работают быстро, программируемы и легко поддерживаются.

Официальный Debian trixie поставляет Neovim 0.10.4, а Bookworm застрял ещё дальше в прошлом. Этот репозиторий предоставляет Neovim 0.12.4 одинаково для Bookworm, Trixie, Forky и Sid.

⚡ Ключевые возможности Neovim

🌙 Конфигурация на Lua

Настраивайте и программируйте редактор на Lua с богатым и стабильным API. Настоящий язык программирования вместо Vimscript для быстрых и легко поддерживаемых конфигураций.

🔌 Встроенный LSP

Нативный клиент Language Server Protocol добавляет автодополнение, диагностику, переход к определению и рефакторинг для любого языка с сервером.

🌳 Treesitter

Инкрементальный разбор для точной подсветки синтаксиса, отступов и структурных выделений — далеко за пределами подсветки на основе регулярных выражений.

🚀 Асинхронность и плагины

Неблокирующий цикл задач и событий сохраняет отзывчивость интерфейса, обеспечивая огромную экосистему современных плагинов на Lua и удалённых интерфейсов.

🚀 Всегда впереди официальных архивов: Debian и Ubuntu замораживают версии пакетов при выпуске релиза — этот репозиторий публикует новые версии обычно в течение нескольких часов после релиза upstream. Простой apt upgrade всегда держит вас на последней версии.

❓ Часто задаваемые вопросы

Есть ли Neovim в официальных репозиториях Debian?

Да, но официальный архив Debian поставляет Neovim 0.10.4, тогда как этот репозиторий предлагает Neovim 0.12.4 — пересобирается через несколько часов после каждого релиза.

Как установить последнюю версию Neovim в Debian?

Добавьте репозиторий deb.griffo.io один раз по инструкции выше, затем выполните: sudo apt install neovim-latest. Новые версии приходят через обычный sudo apt upgrade.

Подписаны ли пакеты и как они собираются?

Каждый пакет подписан GPG-ключом репозитория (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) и собирается из релизов upstream в публичных репозиториях GitHub, которые может проверить любой.
