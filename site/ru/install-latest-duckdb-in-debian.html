Что такое DuckDB?

DuckDB — это быстрая внутрипроцессная аналитическая SQL-база данных — своего рода «SQLite для аналитики». Она работает внутри вызывающего процесса без какого-либо сервера, запрашивает файлы CSV, Parquet и JSON напрямую (SELECT * FROM 'data.parquet') и говорит на богатом, дружелюбном диалекте SQL с отличной производительностью на аналитических нагрузках. Этот пакет поставляет официальный CLI duckdb.

⚠️ Ни сервера, ни демона: DuckDB — встраиваемая база данных: она работает внутри вашего процесса (или CLI), поэтому нет сервиса для настройки, нет порта для открытия и нет юнита systemd. Установите и запустите duckdb. По той же причине один файл базы данных принадлежит одному процессу одновременно.
🚀 Всегда впереди официальных архивов: Debian и Ubuntu замораживают версии пакетов при выпуске релиза — этот репозиторий публикует новые версии обычно в течение нескольких часов после релиза upstream, для каждого поддерживаемого выпуска, включая stable. Простой apt upgrade всегда держит вас на последней версии.

⚡ Ключевые возможности DuckDB

📊 Создана для аналитики

Колоночный векторизованный движок обрабатывает миллионы строк в секунду — агрегации, соединения и оконные функции на скорости OLAP.

📁 Запросы к файлам напрямую

SELECT * FROM 'events.parquet' — CSV, Parquet и JSON можно запрашивать (и записывать) без какого-либо шага импорта.

🪶 Ноль настройки

Внутрипроцессная, как SQLite: один бинарный файл, один необязательный файл базы данных, нечего администрировать.

🔌 Расширения во время выполнения

INSTALL httpfs; подтягивает подписанные расширения из официального репозитория — доступ к S3/HTTP, spatial, полнотекстовый поиск и не только.

🗣️ Дружелюбный SQL

SELECT * EXCLUDE, GROUP BY ALL, типы list/struct, PIVOT — современный диалект, убирающий шаблонный код из повседневных запросов.

🤝 Дружит с другими

Читает SQLite, PostgreSQL и MySQL через расширения; экспортирует Parquet/CSV/JSON — идеальный клей для дата-пайплайнов.

🏆 Зачем устанавливать через apt?

  • Расширения просто работают: этот пакет использует официальную glibc-сборку upstream, поэтому INSTALL httpfs; совместим с официальным репозиторием расширений linux_amd64/linux_arm64
  • Каждый выпуск, включая stable: официальные архивы поставляют DuckDB только в testing/unstable — здесь Bookworm и Trixie получают ту же актуальную версию
  • Без curl-to-bin: репозиторий с GPG-подписью вместо ручной загрузки бинарных файлов в /usr/local/bin
  • Автоматические обновления: новые релизы upstream приходят с обычным apt upgrade
  • Мульти-архитектура: amd64 и arm64 — архитектуры, для которых upstream публикует бинарные файлы CLI под Linux

❓ Часто задаваемые вопросы

Есть ли DuckDB в официальных репозиториях Debian?

Нет — DuckDB не упакован в официальных архивах Debian. Этот репозиторий сейчас единственный источник apt и предлагает DuckDB 1.5.4.

Как установить последнюю версию DuckDB в Debian?

Добавьте репозиторий deb.griffo.io один раз по инструкции выше, затем выполните: sudo apt install duckdb. Новые версии приходят через обычный sudo apt upgrade.

Подписаны ли пакеты и как они собираются?

Каждый пакет подписан GPG-ключом репозитория (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) и собирается из релизов upstream в публичных репозиториях GitHub, которые может проверить любой.

Какие версии Debian поддерживаются?

Debian 12 Bookworm, Debian 13 Trixie, Forky и Sid.
