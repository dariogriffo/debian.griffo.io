Что такое just?

just — это удобный исполнитель команд, написанный на Rust Кейси Родармором. Он сохраняет и запускает специфичные для проекта команды, называемые рецептами, которые хранятся в файле с именем justfile с синтаксисом, вдохновлённым make. В отличие от make, just не является системой сборки: рецепты — это просто команды для выполнения, поэтому здесь нет фиктивных целей (phony) и зависимости от времени изменения файлов. Набрав более 25 тысяч звёзд на GitHub, он стал предпочтительным инструментом запуска задач для современных проектов.

😤 Проблемы традиционного make: Устали от объявлений .PHONY повсюду? Надоели ошибки табуляции против пробелов и неявные правила сборки, которые срабатывают, когда вы меньше всего этого ожидаете? just сохраняет привычное ощущение рецептов Makefile, но избавляется от груза системы сборки — каждый рецепт это просто команда.
🚀 Почему важны последние версии: just активно развивается с частыми релизами, содержащими новые атрибуты, функции выражений, интеграции с оболочками и улучшения удобства использования. Последние версии включают улучшенные скрипты автодополнения, новые встроенные функции, более понятные сообщения об ошибках и расширенную кроссплатформенную поддержку.

⚡ Ключевые возможности just

📝 Синтаксис в стиле make

Пишите рецепты в знакомом, читаемом justfile. Если вы знаете Makefile, вы уже знаете большую часть just — без причуд.

🧩 Параметры рецептов

Рецепты принимают аргументы и имеют значения по умолчанию, поэтому один рецепт может адаптироваться ко многим вызовам без дополнительных скриптов.

🐚 Любая оболочка или язык

Пишите рецепты на sh, bash, Python, Node или чём-либо ещё с помощью строки shebang. just запускает каждый рецепт с выбранным вами интерпретатором.

📋 Список и обнаружение

Запустите just --list, чтобы увидеть каждый рецепт с его комментарием-документацией. Новички могут мгновенно изучить задачи проекта.

🔐 .env и переменные

Загружайте файлы окружения, определяйте переменные и подставляйте их в рецепты. Держите конфигурацию в порядке и вне ваших команд.

🌍 Кроссплатформенность

Единый статический бинарный файл, который одинаково работает на Linux, macOS и Windows — без среды выполнения, без интерпретатора, без причуд.

🏆 Преимущества для производительности

  • Экономия времени: Запускайте любую задачу проекта с помощью just <recipe> вместо запоминания длинных команд
  • Самодокументирование: just --list показывает каждую задачу и её описание
  • Единообразие: Вся команда запускает одни и те же команды одинаково
  • Без причуд сборки: Рецепты — это команды, а не цели — никаких сюрпризов с устаревшими метками времени
  • Переносимость: Один justfile работает на Linux, macOS и Windows
🚀 Всегда впереди официальных архивов: Debian и Ubuntu замораживают версии пакетов при выпуске релиза — этот репозиторий публикует новые версии обычно в течение нескольких часов после релиза upstream. Простой apt upgrade всегда держит вас на последней версии.

# Создать justfile в вашем проекте cat > justfile <<'EOF' # Собрать проект build: cargo build --release # Запустить тесты test: cargo test # Поприветствовать кого-нибудь hello name='world': echo "Hello, {{name}}!" EOF # Показать доступные рецепты just --list # Запустить рецепт just build just hello just

🎯 Примеры базового использования

Запуск рецептов:

# Run the first recipe in the justfile (the default) just # Run a specific recipe just test # Pass an argument to a recipe just hello Dario # List every recipe with its documentation just --list

Работа с justfile:

# Show the resolved value of a variable just --evaluate # Print a recipe without running it just --show build # Use a justfile in another directory just --justfile ~/project/justfile --working-directory ~/project build # Choose a recipe interactively (needs a chooser like fzf) just --choose

Автодополнение оболочки:

# Completions for bash, fish and zsh are installed with the package. # Generate them manually for any shell if you need to: just --completions bash just --completions zsh just --completions fish

❓ Часто задаваемые вопросы

Есть ли just в официальных репозиториях Debian?

Да, но официальный архив Debian поставляет just 1.40.0, тогда как этот репозиторий предлагает just 1.57.0 — пересобирается через несколько часов после каждого релиза.

Как установить последнюю версию just в Debian?

Добавьте репозиторий deb.griffo.io один раз по инструкции выше, затем выполните: sudo apt install just. Новые версии приходят через обычный sudo apt upgrade.

Подписаны ли пакеты и как они собираются?

Каждый пакет подписан GPG-ключом репозитория (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) и собирается из релизов upstream в публичных репозиториях GitHub, которые может проверить любой.

Какие версии Debian поддерживаются?

Debian 12 Bookworm, Debian 13 Trixie, Forky и Sid.
