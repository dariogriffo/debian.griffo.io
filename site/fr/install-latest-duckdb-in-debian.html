Qu'est-ce que DuckDB ?

DuckDB est une base de données SQL analytique in-process et rapide — pensez « SQLite pour l'analytique ». Elle s'exécute dans le processus appelant sans serveur à mettre en place, interroge directement les fichiers CSV, Parquet et JSON (SELECT * FROM 'data.parquet'), et parle un dialecte SQL riche et convivial avec d'excellentes performances sur les charges analytiques. Ce paquet fournit la CLI officielle duckdb.

⚠️ Pas de serveur, pas de démon : DuckDB est une base embarquée — elle s'exécute dans votre processus (ou la CLI), donc aucun service à configurer, aucun port à ouvrir et aucune unité systemd. Installez-la et lancez duckdb. Pour la même raison, un fichier de base appartient à un seul processus à la fois.
🚀 Toujours en avance sur les archives officielles : Debian et Ubuntu figent les versions des paquets à chaque publication — ce dépôt publie les nouvelles versions upstream généralement en quelques heures, pour chaque version supportée, y compris stable. Un simple apt upgrade vous garde toujours à la dernière version.

⚡ Fonctionnalités clés de DuckDB

📊 Conçue pour l'analytique

Un moteur en colonnes et vectorisé qui broie des millions de lignes par seconde — agrégations, jointures et fonctions de fenêtrage à la vitesse OLAP.

📁 Interrogez les fichiers directement

SELECT * FROM 'events.parquet' — CSV, Parquet et JSON sont interrogeables (et inscriptibles) sans aucune étape d'import.

🪶 Zéro configuration

In-process comme SQLite : un unique binaire, un unique fichier de base optionnel, rien à administrer.

🔌 Extensions à l'exécution

INSTALL httpfs; récupère des extensions signées depuis le dépôt officiel — accès S3/HTTP, spatial, recherche plein texte et plus encore.

🗣️ SQL convivial

SELECT * EXCLUDE, GROUP BY ALL, types list/struct, PIVOT — un dialecte moderne qui élimine le boilerplate des requêtes quotidiennes.

🤝 Joue bien avec les autres

Lit SQLite, PostgreSQL et MySQL via des extensions ; exporte Parquet/CSV/JSON — la colle idéale pour les pipelines de données.

🏆 Pourquoi l'installer avec apt ?

  • Les extensions fonctionnent d'emblée : ce paquet utilise le build glibc officiel upstream, donc INSTALL httpfs; correspond au dépôt d'extensions officiel linux_amd64/linux_arm64
  • Chaque version, y compris stable : les archives officielles ne proposent DuckDB qu'en testing/unstable — ici Bookworm et Trixie reçoivent la même version courante
  • Pas de curl-vers-bin : un dépôt signé GPG au lieu de télécharger des binaires à la main dans /usr/local/bin
  • Mises à jour automatiques : les nouvelles versions upstream arrivent avec votre apt upgrade habituel
  • Multi-arch : amd64 et arm64 — les architectures pour lesquelles upstream publie des binaires CLI Linux

❓ Questions fréquentes

DuckDB est-il dans les dépôts officiels de Debian ?

Non — DuckDB n'est pas empaqueté dans les archives officielles de Debian. Ce dépôt est actuellement la seule source apt et propose DuckDB 1.5.5.

Comment installer le dernier DuckDB sur Debian ?

Ajoutez le dépôt deb.griffo.io une seule fois (instructions ci-dessus), puis exécutez : sudo apt install duckdb. Les nouvelles versions arrivent via le sudo apt upgrade habituel.

Les paquets sont-ils signés et comment sont-ils construits ?

Chaque paquet est signé avec la clé GPG du dépôt (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) et construit à partir des versions upstream dans des dépôts GitHub publics que chacun peut inspecter.

Quelles versions de Debian sont prises en charge ?

Debian 12 Bookworm, Debian 13 Trixie, Forky et Sid.
