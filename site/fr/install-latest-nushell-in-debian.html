Qu'est-ce que Nushell ?

Nushell (la commande nu) est un shell moderne construit autour des données structurées. Au lieu de faire transiter du texte brut entre les commandes, chaque pipeline transporte des tables et des enregistrements typés — ls renvoie des lignes que vous pouvez filtrer, trier et additionner, et JSON, YAML, CSV, TOML et SQLite sont des citoyens de première classe. Il combine un vrai langage de programmation avec un shell interactif, offrant d'emblée des messages d'erreur clairs, l'autocomplétion et la coloration syntaxique.

⚠️ Prêt à servir de shell de connexion : ce paquet enregistre nu dans /etc/shells à l'installation (et le retire proprement à la désinstallation), donc chsh -s /usr/bin/nu fonctionne immédiatement — aucune édition manuelle. Le binaire est lié statiquement (musl), votre shell continue donc de fonctionner quelles que soient les mises à jour de bibliothèques.
🚀 Toujours en avance sur les archives officielles : Debian et Ubuntu figent les versions des paquets à chaque publication — ce dépôt publie les nouvelles versions upstream généralement en quelques heures. Un simple apt upgrade vous garde toujours à la dernière version.

⚡ Fonctionnalités clés de Nushell

📊 Pipelines structurés

Les commandes émettent des tables, pas du texte : ls | where size > 1mb | sort-by modified — sans gymnastique awk, cut ou sed.

🔄 Tous les formats de données

Ouvrez et transformez JSON, YAML, CSV, TOML, XML et SQLite nativement : open Cargo.toml | get package.version.

🧠 Un vrai langage

Variables typées, closures, commandes personnalisées et modules — avec des messages d'erreur dignes d'un compilateur qui pointent l'emplacement exact.

🔌 Plugins inclus

Fournit les plugins officiels : polars (dataframes), query (JSON/XML/web scraping), formats, gstat et inc.

🔐 Prêt pour le shell de connexion

Enregistré dans /etc/shells par le paquet — basculez avec chsh -s /usr/bin/nu dès qu'apt a terminé.

📦 Zéro dépendance

Un unique binaire statique musl. Pas de runtime, pas de bibliothèques susceptibles de casser — idéal pour un programme qui vous sert de session.

🏆 Pourquoi l'installer avec apt ?

  • Sûr pour un shell : builds liés statiquement, enregistrés et désenregistrés dans /etc/shells par les scripts du paquet — les détails qui comptent quand il s'agit de votre shell de connexion
  • Plugins inclus : le jeu de plugins officiel est fourni dans le même paquet, aligné sur la version exacte de nu (les plugins l'exigent)
  • Pas de curl-vers-bin : un dépôt signé GPG au lieu de télécharger des binaires à la main dans /usr/local/bin
  • Mises à jour automatiques : les nouvelles versions upstream arrivent avec votre apt upgrade habituel — important, Nushell publiant une version chaque mois
  • Multi-arch : amd64, arm64, armhf et riscv64

❓ Questions fréquentes

Nushell est-il dans les dépôts officiels de Debian ?

Non — Nushell n'est pas empaqueté dans les archives officielles de Debian. Ce dépôt est actuellement la seule source apt et propose Nushell 0.114.1.

Comment installer le dernier Nushell sur Debian ?

Ajoutez le dépôt deb.griffo.io une seule fois (instructions ci-dessus), puis exécutez : sudo apt install nushell. Les nouvelles versions arrivent via le sudo apt upgrade habituel.

Les paquets sont-ils signés et comment sont-ils construits ?

Chaque paquet est signé avec la clé GPG du dépôt (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) et construit à partir des versions upstream dans des dépôts GitHub publics que chacun peut inspecter.

Quelles versions de Debian sont prises en charge ?

Debian 12 Bookworm, Debian 13 Trixie, Forky et Sid.
