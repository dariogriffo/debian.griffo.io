Qu'est-ce que Neovim ?

Neovim est un éditeur de texte extensible basé sur Vim qui refactorise le code historique de Vim pour le développement moderne. Il embarque un client du protocole Language Server intégré, une analyse de la syntaxe et du code propulsée par Treesitter, un moteur de tâches asynchrone et un scripting Lua de premier ordre — pour des plugins et une configuration rapides, programmables et faciles à maintenir.

Debian trixie officiel fournit Neovim 0.10.4, et Bookworm est bloqué sur une version bien plus ancienne. Ce dépôt propose Neovim 0.12.4 sur Bookworm, Trixie, Forky et Sid indifféremment.

⚡ Fonctionnalités clés de Neovim

🌙 Configuration en Lua

Configurez et scriptez l'éditeur en Lua avec une API riche et stable. Un véritable langage de programmation à la place de Vimscript, pour des configurations rapides et faciles à maintenir.

🔌 LSP intégré

Un client natif du protocole Language Server apporte complétion, diagnostics, aller à la définition et refactorisation à tout langage disposant d'un serveur.

🌳 Treesitter

Analyse incrémentale pour une coloration syntaxique, une indentation et des sélections structurelles précises — bien au-delà de la coloration fondée sur des expressions régulières.

🚀 Asynchrone & plugins

Une boucle d'événements et de tâches non bloquante garde l'interface réactive et alimente un vaste écosystème de plugins Lua modernes et d'interfaces distantes.

🚀 Toujours en avance sur les archives officielles : Debian et Ubuntu figent les versions des paquets à chaque publication — ce dépôt publie les nouvelles versions upstream généralement en quelques heures. Un simple apt upgrade vous garde toujours à la dernière version.

❓ Questions fréquentes

Neovim est-il dans les dépôts officiels de Debian ?

Oui, mais l'archive officielle de Debian fournit Neovim 0.10.4, alors que ce dépôt propose Neovim 0.12.4, reconstruit quelques heures après chaque version upstream.

Comment installer le dernier Neovim sur Debian ?

Ajoutez le dépôt deb.griffo.io une seule fois (instructions ci-dessus), puis exécutez : sudo apt install neovim-latest. Les nouvelles versions arrivent via le sudo apt upgrade habituel.

Les paquets sont-ils signés et comment sont-ils construits ?

Chaque paquet est signé avec la clé GPG du dépôt (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) et construit à partir des versions upstream dans des dépôts GitHub publics que chacun peut inspecter.
