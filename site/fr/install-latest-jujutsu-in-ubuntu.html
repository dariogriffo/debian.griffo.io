Qu'est-ce que Jujutsu ?

Jujutsu est un système de contrôle de version compatible avec Git qui offre un modèle plus simple et plus puissant que Git tout en restant compatible avec les dépôts Git existants. Il utilise la commande jj, traite la copie de travail comme un commit, offre une gestion des conflits de premier ordre et une annulation pour chaque opération — vous permettant de continuer à utiliser un hébergement Git comme GitHub ou Forgejo en dessous.

🚀 Pourquoi les dernières versions sont importantes : Jujutsu évolue rapidement, avec de nouvelles commandes, des améliorations de workflow et des gains de performance. Utiliser la dernière version signifie avoir accès aux fonctionnalités les plus récentes et aux correctifs d'interopérabilité avec Git.

⚡ Fonctionnalités clés de Jujutsu

🔄 Compatible avec Git

Fonctionne directement avec les dépôts Git existants et des services distants comme GitHub et Forgejo.

↩️ Annulation universelle

Chaque opération est enregistrée et peut être annulée avec une seule commande.

🧩 Conflits de premier ordre

Les conflits sont enregistrés dans les commits et peuvent être résolus à tout moment, pas seulement pendant les fusions.

📄 Copie de travail en tant que commit

La copie de travail est elle-même un commit, ce qui supprime la zone de préparation (staging) et simplifie les workflows.

🎯 Pourquoi Jujutsu ?

  • Modèle plus simple : Pas de zone de préparation, et un workflow cohérent et prévisible
  • Expérimentation sans risque : Annulez n'importe quelle opération, y compris les rebases et les fusions
  • Interopérabilité avec Git : Continuez à utiliser l'hébergement Git et collaborez avec des utilisateurs de Git
  • Édition d'historique puissante : Rebasage et réécriture de commits sans effort

🆕 Points forts

  • La commande jj : Une CLI moderne au-dessus des dépôts Git
  • Journal des opérations : Historique complet des opérations sur le dépôt, avec annulation
  • Gestion des conflits : Conflits stockés dans les commits, résolvables à tout moment
  • Backend Git : Interopère avec les dépôts Git distants existants
