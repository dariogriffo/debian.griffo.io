Was ist Neovim?

Neovim ist ein hyper-erweiterbarer, Vim-basierter Texteditor, der die klassische Vim-Codebasis für die moderne Entwicklung überarbeitet. Er bringt einen integrierten Language-Server-Protocol-Client, Treesitter-gestützte Syntax- und Code-Analyse, eine asynchrone Job-Engine und erstklassiges Lua-Skripting mit — so werden Plugins und Konfiguration schnell, programmierbar und wartbar.

Das offizielle Debian trixie liefert Neovim 0.10.4, und Bookworm hängt noch deutlich weiter zurück. Dieses Repository stellt Neovim 0.12.4 für Bookworm, Trixie, Forky und Sid gleichermaßen bereit.

⚡ Wichtigste Funktionen von Neovim

🌙 Lua-Konfiguration

Konfigurieren und skripten Sie den Editor in Lua mit einer umfangreichen, stabilen API. Eine echte Programmiersprache statt Vimscript für schnelle, wartbare Setups.

🔌 Integriertes LSP

Ein nativer Language-Server-Protocol-Client bringt Autovervollständigung, Diagnosen, Sprung-zur-Definition und Refactoring in jede Sprache mit einem Server.

🌳 Treesitter

Inkrementelles Parsing für präzise Syntaxhervorhebung, Einrückung und strukturelle Selektionen — weit über regex-basierte Hervorhebung hinaus.

🚀 Async & Plugins

Eine nicht-blockierende Job- und Event-Loop hält die Oberfläche reaktionsfähig und treibt ein riesiges Ökosystem moderner Lua-Plugins und Remote-UIs an.

🚀 Immer aktueller als die offiziellen Archive: Debian und Ubuntu frieren Paketversionen mit jedem Release ein — dieses Repository veröffentlicht neue Upstream-Versionen in der Regel innerhalb weniger Stunden. Ein einfaches apt upgrade hält Sie immer auf der neuesten Version.

❓ Häufig gestellte Fragen

Ist Neovim in den offiziellen Debian-Paketquellen?

Ja, aber das offizielle Debian-Archiv liefert Neovim 0.10.4, während dieses Repository Neovim 0.12.4 bereitstellt — wenige Stunden nach jedem Upstream-Release neu gebaut.

Wie installiere ich das neueste Neovim unter Debian?

Fügen Sie das deb.griffo.io-Repository einmalig hinzu (Anleitung oben) und führen Sie dann aus: sudo apt install neovim-latest. Neue Releases kommen über das normale sudo apt upgrade.

Sind die Pakete signiert und wie werden sie gebaut?

Jedes Paket ist mit dem GPG-Schlüssel des Repositories (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) signiert und wird aus Upstream-Releases in öffentlichen GitHub-Repositories gebaut, die jeder einsehen kann.
