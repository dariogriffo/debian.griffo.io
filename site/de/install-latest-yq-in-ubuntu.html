Was ist yq?

yq ist ein leichtgewichtiger, portabler Kommandozeilen-Prozessor für strukturierte Daten — YAML, JSON, XML, CSV, TOML und Properties-Dateien — geschrieben in Go von Mike Farah. Es nutzt jq-ähnliche Syntax, sodass sich alles, was Sie von jq kennen, direkt übertragen lässt: Dateien abfragen, transformieren, zusammenführen und direkt bearbeiten, in Skripten und Pipelines, mit einem einzigen abhängigkeitsfreien Binary.

⚠️ Welches yq ist das? Dies ist mikefarah/yq (v4.x, Go) — das yq aus den meisten CI-Beispielen und Dokumentationen. Es ist nicht der Python-basierte yq jq-Wrapper aus manchen Archiven; dieses Paket ersetzt ihn sauber, und weder python noch jq werden benötigt.
🚀 Immer aktueller als die offiziellen Archive: Debian und Ubuntu frieren Paketversionen mit jedem Release ein — dieses Repository veröffentlicht neue Upstream-Versionen in der Regel innerhalb weniger Stunden. Ein einfaches apt upgrade hält Sie immer auf der neuesten Version.

⚡ Hauptfunktionen von yq

🔄 Sechs Formate

YAML, JSON, XML, CSV/TSV, TOML und Properties — eines lesen, ein anderes ausgeben: yq -o=json '.' config.yaml konvertiert in einem Schritt.

🧠 jq-ähnliche Syntax

Vertraute Pfade, Pipes, Funktionen und Operatoren. Wer einen jq-Filter schreiben kann, kann yq sofort auf YAML anwenden.

✏️ In-Place-Bearbeitung

yq -i '.spec.replicas = 3' deploy.yaml — skriptgesteuerte Änderungen an Kubernetes-Manifesten, CI-Konfigurationen und Compose-Dateien, Kommentare bleiben erhalten.

📦 Null Abhängigkeiten

Ein einzelnes statisches Go-Binary. Keine Runtime, keine Bibliotheken, kein python — ideal für minimale Container und CI-Runner.

🔀 Merge & Anchors

Führt mehrere Dateien tief zusammen, löst YAML-Anchors und -Aliasse auf, verarbeitet Multi-Dokument-Streams — die schwierigen YAML-Fälle korrekt gelöst.

📖 Man-Page enthalten

Dieses Paket liefert die vollständige Man-Page zusammen mit dem Binary — man yq funktioniert offline, ganz nach Debian-Konvention.

🏆 Warum mit apt installieren?

  • Flotten-tauglich: ein Repo in Ihren Basis-Images, und jeder Server, Container und CI-Runner erhält dasselbe aktuelle yq
  • Kein curl-to-bin: ein GPG-signiertes Repository statt Binaries von Hand nach /usr/local/bin herunterzuladen
  • Das richtige yq: garantiert überall mikefarah/yq v4-Semantik — keine Überraschungen durch den Python-Wrapper auf manchen Maschinen
  • Automatische Updates: neue Upstream-Releases kommen mit Ihrem normalen apt upgrade
  • Multi-Arch: amd64, arm64, armhf, ppc64el, s390x und riscv64

Ist yq in den offiziellen Ubuntu-Paketquellen?

Nein — yq ist nicht in den offiziellen Ubuntu-Archiven paketiert. Dieses Repository ist derzeit die einzige apt-Quelle und liefert yq 4.53.3.

Wie installiere ich das neueste yq unter Ubuntu?

Fügen Sie das deb.griffo.io-Repository einmalig hinzu (Anleitung oben) und führen Sie dann aus: sudo apt install yq. Neue Releases kommen über das normale sudo apt upgrade.

Sind die Pakete signiert und wie werden sie gebaut?

Jedes Paket ist mit dem GPG-Schlüssel des Repositories (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) signiert und wird aus Upstream-Releases in öffentlichen GitHub-Repositories gebaut, die jeder einsehen kann.
