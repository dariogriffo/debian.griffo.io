Was ist DuckDB?

DuckDB ist eine schnelle, analytische In-Process-SQL-Datenbank — quasi "SQLite für Analytics". Sie läuft im aufrufenden Prozess, ganz ohne Server-Setup, fragt CSV-, Parquet- und JSON-Dateien direkt ab (SELECT * FROM 'data.parquet') und spricht einen reichhaltigen, freundlichen SQL-Dialekt mit exzellenter Performance bei analytischen Workloads. Dieses Paket liefert das offizielle duckdb-CLI.

⚠️ Kein Server, kein Daemon: DuckDB ist eine eingebettete Datenbank — sie läuft in Ihrem Prozess (oder im CLI), es gibt also keinen Dienst zu konfigurieren, keinen Port zu öffnen und keine systemd-Unit. Installieren und duckdb starten. Aus demselben Grund gehört eine Datenbankdatei immer nur einem Prozess gleichzeitig.
🚀 Immer aktueller als die offiziellen Archive: Debian und Ubuntu frieren Paketversionen mit jedem Release ein — dieses Repository veröffentlicht neue Upstream-Versionen in der Regel innerhalb weniger Stunden, für jedes unterstützte Release inklusive der LTS-Versionen. Ein einfaches apt upgrade hält Sie immer auf der neuesten Version.

⚡ Hauptfunktionen von DuckDB

📊 Gebaut für Analytics

Eine spaltenorientierte, vektorisierte Engine verarbeitet Millionen Zeilen pro Sekunde — Aggregationen, Joins und Window-Funktionen mit OLAP-Geschwindigkeit.

📁 Dateien direkt abfragen

SELECT * FROM 'events.parquet' — CSV, Parquet und JSON sind ohne Importschritt abfragbar (und beschreibbar).

🪶 Null Setup

In-Process wie SQLite: ein einzelnes Binary, eine einzelne optionale Datenbankdatei, nichts zu administrieren.

🔌 Runtime-Extensions

INSTALL httpfs; lädt signierte Extensions aus dem offiziellen Repository — S3/HTTP-Zugriff, Spatial, Volltextsuche und mehr.

🗣️ Freundliches SQL

SELECT * EXCLUDE, GROUP BY ALL, List/Struct-Typen, PIVOT — ein moderner Dialekt, der Boilerplate aus alltäglichen Abfragen entfernt.

🤝 Spielt gut mit anderen

Liest SQLite, PostgreSQL und MySQL über Extensions; exportiert Parquet/CSV/JSON — idealer Klebstoff für Daten-Pipelines.

🏆 Warum mit apt installieren (statt Snap oder manuellem Download)?

  • Extensions funktionieren einfach: dieses Paket verwendet Upstreams offiziellen glibc-Build, sodass INSTALL httpfs; zum offiziellen linux_amd64/linux_arm64-Extension-Repository passt
  • Jedes Release, inklusive der LTS-Versionen: die offiziellen Ubuntu-Archive liefern DuckDB nicht — hier erhalten Jammy bis Resolute alle dieselbe aktuelle Version
  • Kein curl-to-bin: ein GPG-signiertes Repository statt Binaries von Hand nach /usr/local/bin herunterzuladen
  • Automatische Updates: neue Upstream-Releases kommen mit Ihrem normalen apt upgrade
  • Multi-Arch: amd64 und arm64 — die Architekturen, für die Upstream Linux-CLI-Binaries veröffentlicht

❓ Häufig gestellte Fragen

Ist DuckDB in den offiziellen Ubuntu-Paketquellen?

Nein — DuckDB ist nicht in den offiziellen Ubuntu-Archiven paketiert. Dieses Repository ist derzeit die einzige apt-Quelle und liefert DuckDB 1.5.4.

Wie installiere ich das neueste DuckDB unter Ubuntu?

Fügen Sie das deb.griffo.io-Repository einmalig hinzu (Anleitung oben) und führen Sie dann aus: sudo apt install duckdb. Neue Releases kommen über das normale sudo apt upgrade.

Sind die Pakete signiert und wie werden sie gebaut?

Jedes Paket ist mit dem GPG-Schlüssel des Repositories (EA0F721D231FDD3A0A17B9AC7808B4DD62C41256) signiert und wird aus Upstream-Releases in öffentlichen GitHub-Repositories gebaut, die jeder einsehen kann.

Welche Ubuntu-Versionen werden unterstützt?

Ubuntu 22.04 Jammy, 24.04 Noble, 25.10 Questing und 26.04 Resolute.
