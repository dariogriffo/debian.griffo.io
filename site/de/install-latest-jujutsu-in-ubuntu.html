Was ist Jujutsu?

Jujutsu ist ein Git-kompatibles Versionskontrollsystem, das ein einfacheres und leistungsfähigeres Modell als Git bietet und dabei vollständig abwärtskompatibel zu bestehenden Git-Repositories bleibt. Es verwendet den Befehl jj, behandelt die Arbeitskopie als Commit, bietet erstklassige Konflikthandhabung und ein Undo für jede Operation — sodass Sie weiterhin Git-Hosting wie GitHub oder Forgejo darunter nutzen können.

🚀 Warum neueste Versionen wichtig sind: Jujutsu entwickelt sich rasant weiter, mit neuen Befehlen, Workflow-Verfeinerungen und Performance-Verbesserungen. Mit der neuesten Version erhalten Sie Zugriff auf die neuesten Funktionen und Git-Interop-Fixes.

⚡ Wichtigste Funktionen von Jujutsu

🔄 Git-kompatibel

Funktioniert direkt mit bestehenden Git-Repositories und Remotes wie GitHub und Forgejo.

↩️ Universelles Undo

Jede Operation wird aufgezeichnet und kann mit einem einzigen Befehl rückgängig gemacht werden.

🧩 Erstklassige Konflikte

Konflikte werden in Commits gespeichert und können jederzeit gelöst werden, nicht nur bei Merges.

📄 Arbeitskopie als Commit

Die Arbeitskopie ist selbst ein Commit, wodurch die Staging-Area entfällt und Workflows vereinfacht werden.

🎯 Warum Jujutsu?

  • Einfacheres Modell: Keine Staging-Area und ein konsistenter, vorhersehbarer Workflow
  • Sicheres Experimentieren: Jede Operation rückgängig machen, einschließlich Rebases und Merges
  • Git-Interop: Weiterhin Git-Hosting nutzen und mit Git-Nutzern zusammenarbeiten
  • Leistungsstarke Historienbearbeitung: Müheloses Rebasing und Umschreiben von Commits

🆕 Highlights

  • Der jj-Befehl: Eine moderne CLI für Git-Repositories
  • Operation Log: Vollständige Historie der Repository-Operationen mit Undo
  • Konflikthandhabung: Konflikte werden in Commits gespeichert und sind jederzeit lösbar
  • Git-Backend: Interoperiert mit bestehenden Git-Remotes
